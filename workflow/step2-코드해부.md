# STEP 2. 코드 해부 — "재료가 뭐가 있지"

> **트랙 분기**: `track=code`면 아래 코드 해부를 진행. `track=concept`면 맨 끝 "개념 해부(개념 트랙)" 섹션으로 간다.

> **목적**: 완성 코드를 의도 필터로 분석. 핵심 기능 추출.
> **전제**: 코드 워크플로우 산출물이 code/ 폴더에 있거나 GitHub URL 제공.
> **스킬**: A1.구조-스캐너, A2.기능-추출기, A3.기술스택-탐지기, A4.의존성-매퍼
> **검토**: [인사이트 검토] + [감수 검토]
> **산출물**: `planning/code-analysis.md`

## 흐름

```
--- AI 자동 분석 ---
1. A1.구조-스캐너 → 프로젝트 트리
2. A2.기능-추출기 → 기능 목록
3. A3.기술스택-탐지기 → 기술 스택
4. A4.의존성-매퍼 → 컴포넌트 의존성

--- 의도 필터 적용 ---
5. planning/seed.md 의도 필터 적용 → 의도 안/밖 분류
6. "이 기능 목록 맞아?" → 핵심 기능 확인
7. "이 기능 중에 안 보여줘도 되는 건?" → 범위 확정
    ↓
[인사이트 검토] → 코드에서 발견한 이슈 질문
    ↓
[감수 검토]
  - 기술 감수자: 빠진 핵심 개념은? 기술 의존성은?
  - 독자 대변인: 첫 버전이 너무 어렵지 않나?
    ↓
저자 확인 → planning/code-analysis.md 생성
```

## 개념 해부 (개념 트랙)

> **목적**: 목차를 의도 필터로 분석. **산출물**: `planning/concept-analysis.md`

### 분석 항목
1. **강별 핵심 개념**: 각 강에서 독자가 꼭 쥐어야 할 한 개념.
2. **선수 개념(의존성) 맵**: 강 A를 이해하려면 어떤 강이 먼저인가. 목차의 "의존 관계" 표가 있으면 그대로 흡수.
3. **비유 ↔ 정식 정의 쌍**: 비유 후크와 그것이 가리키는 정식 수학 정의를 짝짓는다.
4. **의도 밖 가지치기**: 의도(seed) 밖이면 제외. (예: ε-δ 엄밀증명, 증명 디테일)

### 산출물 템플릿: planning/concept-analysis.md
```markdown

## 강별 핵심 개념
| 강 | 핵심 개념 | 비유 후크 | 정식 정의(요약) |
|----|----------|----------|----------------|
| 1강 | 포물선 꼭짓점=최솟값 | 공 던지기 | f(x)=ax²+bx+c의 최솟값 |

## 의존성 맵
| 강 | 선수 강 | 비고 |
|----|---------|------|

| 주제 | 제외 이유 |
|------|----------|
```
