## 한 챕터의 흐름

```
--- 선수 확인 (집필 전, 필수) ---
0. **planning/known.md 읽기**. "누적 지식 원장"에서 독자가 지금 아는 개념·기호를 확인하고,
   "앞으로의 선수 체인"에서 이 챕터가 뒤 챕터 개념을 당겨쓰지 않는지 점검한다.
   → 이 챕터에서 쓸 용어·기호를 분류: [known=그냥 씀] / [모름+핵심=이 챕터서 처음부터 설명] / [모름+곁가지=빼거나 이름만] / [뒤 챕터 것=금지]

--- 변수 수집 ---
1~8번 질문 → [인사이트 검토]: 놓친 부분 질문

--- 이야기 파트 작성 ---

10b. **비유가 나오는 곳에 일러스트** 삽입(그림이 몰입을 돕는 자리에만, 장수 채우기 아님). `[IMAGE PROMPT]` 블록 형식. 이야기 파트는 코드만 금지이며 그림은 권장.
```

## 챕터 내부 구조 (상수)

```
┌─────────────────────────────────────┐
│  이야기 파트 (코드 없음)              │
│  - 도입 (경험, 상황)                 │
│  - 개념 (비유 1~2문장)               │
│  - 시나리오 진행 (이번 버전에서 뭘 바꾸나)│
│  - "이것만은 기억하자" + 다음 예고     │
├─────────────────────────────────────┤
│  기술 파트 (이야기 끝난 후)            │
│  - 용어 정리 (비유→정식 정의)          │
│  - 파일 계층 구조 [실습/설명/참고]      │
│  - 실습 코드 + 설명 코드              │
│  - 더 알아보기                       │
└─────────────────────────────────────┘
```

---

> **트랙 분기**: 이야기 파트는 양 트랙 공유. 기술 파트만 분기한다.
> - `code`: 용어정리 + 파일계층 + 실습/설명 코드 (기존)
> - `concept`: 아래 "기술 파트(개념 트랙)" 구조.
> - `lecture`: 아래 "기술 파트 (강의 트랙)" 구조. 한 챕터의 원료 = 해당 강의 판서대본 + 자막 + (있으면) 시뮬레이터 캡처.

### 기술 파트 (개념 트랙)
순서대로 작성한다. `math` 스킬을 로드한다.
1. **용어 정리**: 비유 → 정식 정의(LaTeX).
2. **수식 전개**: 핵심 수식을 LaTeX로. `$인라인$` / `$$블록$$`.
3. **계산 예제**: examples/의 worked example을 단계별로.
4. **다이어그램/그래프**: visual 스킬(Mermaid) 또는 이미지 플레이스홀더. 이미지 플레이스홀더는 **반드시** visual 스킬 `references/image.md`의 블록 형식을 따른다: `<!-- [IMAGE PROMPT: id]` / `path: assets/CH{N}/...png` / `-->` / `![](../assets/CH{N}/...png)`. 맨 `[IMAGE PROMPT]` 한 줄만 쓰면 `image_gen.py`가 감지하지 못한다. 백엔드는 Codex(GPT) CLI.
   - **이미지(필요한 곳에)**: 직관이 필요한 핵심 개념·수식/그래프에 도식을 붙인다 — **장수 쿼터 아님**(글로 충분하면 생략, 도움 되면 충분히). 흐름·구조는 Mermaid, 비유·기하·그래프는 `[IMAGE PROMPT]`.
5. **연습문제**: 2~3개. 해답은 부록(STEP 7)에 모은다.
6. **"이게 AI 어디에 쓰이나" 한 줄**.

이야기 파트의 "이번 버전에서 뭘 바꾸나"는 개념 트랙에서 "이번 강에서 뭘 새로 쌓나"로 읽는다.

### 기술 파트 (강의 트랙)

1. **용어 정리**: 이야기 파트에서 비유로 넘긴 용어를 정식 정의한다(비유 → 정의 필수).
2. **원리·근거**: "왜 그런가"를 강의 흐름대로 설명한다. 비유로 끝내지 말고 원리·근거(좋은 책·논문·인지심리)를 녹인다.
3. **자막 디테일 살리기**: 자막에서 강사가 즉흥으로 든 실제 예시·표현을 본문에 살린다(필러어는 버린다).
4. **(필요 시) 교육용 코드/수식**: 강의에 코드·수식이 있으면 기술 파트에서만, 교육용으로 제시한다.
5. **자료 삽입**: 슬라이드·판서·시뮬레이터 화면을 `![설명](경로)` 또는 `[CAPTURE NEEDED]`로 배치한다.
6. **더 알아보기**: 해당 강 주제의 심화 갈래를 한두 줄로 안내한다.

## 제목 형식 규칙

목차에 표시되는 제목(챕터 제목, 소제목)은 모두 아래 형식을 따른다:

```
[핵심어] : [독자가 알게 될 것]
```

- **핵심어**: 해당 섹션의 중심 개념/키워드 (기술 용어가 아니어도 됨)
- **설명**: 독자가 이 섹션을 읽고 나서 "아, 이거구나" 하는 것을 한 줄로
- 예시: `말콤 맥린 : 컨테이너의 아버지`, `docker run : 컨테이너를 실행하는 방법`, `가상화 : 하나의 컴퓨터를 여러 개처럼`

**적용 범위**: 챕터 제목(`#`), 소제목(`##`, `###`) — 목차에 표시되는 수준까지
**제외**: `## 이야기 파트`, `## 기술 파트` 같은 구조 구분 레이블, `### 이것만은 기억하자` 같은 고정 섹션
