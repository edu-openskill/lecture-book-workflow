# STEP 3. 챕터 매핑 — "강을 어떤 순서·묶음으로 책에 담지"

> **목적**: 강(講)을 책의 부(Part)/장(Chapter)으로 배열한다. (원본의 "코드 버전 분해"를 "개념 누적 구성"으로 대체)
> **전제**: `planning/lecture-analysis.md` 완성.
> **스킬**: `planning`(분량 계산), `lecture`(L3 의존성 그래프)
> **검토**: [인사이트 검토] + [감수 검토]
> **산출물**: `planning/chapter-map.md`

---

## 원본과 다른 점

| 원본(book-workflow) | 이 하네스 |
|---|---|
| 코드를 버전(v0.1→v0.2…)으로 쪼갬 | 강을 부/장으로 배열 |
| 챕터 = 코드 한 버전 | 장 = 강 하나 (1강 = 1장) |
| 성장 = 코드가 자람 | 성장 = 개념이 누적됨 |

기본 규칙: **한 강 = 한 장.** 강의 막(ACT)이 책의 부(Part)가 된다. 묶거나 쪼갤 필요가 있으면 STEP 3에서 결정.

---

## 흐름

```
1. lecture-analysis의 의존성 그래프 + 목차 막 구조 읽기
2. 막(ACT) → 부(Part) 매핑, 강 → 장 매핑 (기본 1:1)
3. 묶기/쪼개기 판단:
   - 너무 짧은 강 여럿 → 한 장으로 묶을지
   - 너무 긴 강 → 두 장으로 나눌지
4. 각 부 첫머리 "이 부에서 풀 문제" 한 단락(브릿지) 자리 잡기
5. planning(분량 계산)으로 부/장별 예상 페이지 추산
    ↓
[인사이트 검토] → 순서·묶음에서 독자가 막힐 지점 질문
    ↓
[감수 검토] (이야기 편집장: 부 사이 흐름이 자연스러운가)
    ↓
저자 확인 → planning/chapter-map.md 생성
```
