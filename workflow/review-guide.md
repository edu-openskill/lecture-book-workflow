## 1. 인사이트 검토

> **목적**: 저자가 놓치고 있는 부분을 짚어주는 추가 질문

### 체크리스트

```
□ 저자 답변에서 모호하거나 구체화가 필요한 부분이 있는가?
□ 코드 분석 결과와 저자 답변 사이에 충돌이 있는가?
□ 이전 STEP 답변과 현재 답변 사이에 모순이 있는가?
□ 분량(100p 권장) 관점에서 현재 범위가 현실적인가?
□ 독자 관점에서 빠진 선행 개념이 있는가?
□ 저자가 아직 결정하지 않은 채 넘어간 부분이 있는가?
```

### 질문 관리

- 인사이트 질문은 `questions/pending/`에 파일로 저장
- 저자가 답변하면 `questions/done/`으로 이동
- 답변 시 "어떤 산출물에 어떻게 반영했는지" 기록

## 2. 의도감시 검토

> **목적**: 챕터가 의도에서 벗어나지 않았는지 검증
> **참조**: planning/seed.md

### 체크리스트 (7항목)

```
□ 의도 일치     이 챕터의 내용이 "책의 의도"에 부합하는가?
□ 범위 이탈     "의도 밖 범위"에 해당하는 내용이 슬며시 들어오지 않았는가?
□ 코드 성격     예제 코드가 교육용인가? (실무 코드가 아닌가?)
□ 깊이 적절     이 책의 목적(이해 vs 구현)에 맞는 깊이인가?
□ 비유 독립     이야기 파트의 비유가 기술 파트 없이도 이해되는가?
□ 핵심 유지     챕터 핵심 메시지가 책 전체 핵심 메시지와 연결되는가?
□ 상수 준수     챕터 내부 구조(이야기→기술), 비유 길이, 코드 분류를 지켰는가?
```

### 출력 형식

```
✅ 의도 일치: OK — "..."
⚠️ 범위 이탈 감지: → [위치]에서 [내용]. 제안: [수정 방향]
⚠️ 코드 성격 경고: → [위치]에서 [내용]. 제안: [수정 방향]
✅ 깊이 적절: OK
...
```

## 2.5 독자 지식 검증 (known.md 대조) — STEP 5

> **목적**: 챕터가 "독자가 지금 아는 것"의 범위를 벗어나지 않았는지 기계적으로 대조.
> **참조**: `planning/known.md` (독자 지식 원장)
> **트랙**: 개념·수학 트랙에서 특히 중요(선수 누적 구조). 코드·강의 트랙도 권장.

### 체크리스트 (나) 섹션 = 저자가 준 8문항 + 선수 2문항)

```
□ [정보충족] 필요한 정보가 다 있으면서 쉬운가?
□ [순차이해] 한 줄씩 따라가며 점프 없이 이해되는가?
□ [목적도달] 초보자가 천천히 읽으면 이 장의 목적에 도달하는가?
□ [유도완결] 수식 유도과정이 있는가? "다 안다고 치고" 중간을 생략하지 않았는가?
□ [생략점검] 짧게 쓰려다 꼭 필요한 내용을 빠뜨리지 않았는가?
□ [사실검증] 오늘 날짜 기준 자료로 내용이 맞는지 검증했는가? 더 쉬운 설명을 참고했는가?
□ [계산이미지] 계산 예제에 이미지를 붙이면 더 쉬운데, 할 수 있는 걸 안 한 곳은 없는가?
□ [어려움시각화] 어려운 부분에 이미지를 붙여 시각화했는가?
□ ★[선수위반] known.md 원장에 없는 용어·기호를 정의 없이 썼는가? (썼다면 위반 → 정의 삽입 또는 제거)
□ ★[미래당김] outline상 뒤 챕터의 개념·기호를 미리 꺼내 쓰지 않았는가?
```

> ★ 두 줄이 핵심 — "편미분이 정의 없이 등장", "벡터를 벡터 강 전에 사용" 류를 잡는다.
> 통과하면 → known.md 원장에 이 챕터 줄을 추가(append-only)한다.

### 출력 형식

```
✅ [선수위반] OK — 새 용어 '편미분'은 이 챕터 내에서 비유→정의→손계산으로 처음부터 설명함
⚠️ [미래당김] 감지 — [위치]에서 '내적'(Ch.11) 사용. 제안: 직관 그림으로 대체하고 "11강에서" 보류
⚠️ [유도완결] 경고 — d/dx ln x = 1/x 를 단언만 함. 제안: 유도 3단 삽입
```

## 3. 감수 검토 (3인 편집장)

> **목적**: 3가지 관점에서 질문으로 감수

### 3인 편집장

| 편집장 | 관점 | 핵심 질문 스타일 |
|--------|------|-----------------|
| **기술 감수자** | 기술 정확성, 빠진 개념, 순서 | "독자가 OO를 모르는데 여기서 OO이 나와도 될까?" |
| **독자 대변인** | 난이도 곡선, 진입장벽, 완주 가능성 | "여기서 독자가 포기하지 않을까?" |
| **이야기 편집장** | 서사 흐름, 비유 적절성, 몰입도 | "이 도입이 설명으로 시작하는데, 경험담으로 열면 어떨까?" |

> **개념 트랙**: "기술 감수자"를 **수학 감수자**로 읽는다.
> - 수식·정의가 정확한가? 표기가 일관된가(같은 기호 = 같은 뜻)?
> - 의존성 순서가 맞는가(선수 개념이 먼저 나왔나)?
> - 비유 → 정식 정의 전환이 실제로 이뤄졌나?

### 동작

```
산출물 완성 → 3인 편집장이 각자 2~3개 질문 → 저자가 답하며 수정 여부 결정
```

> "넘어가자"도 가능
