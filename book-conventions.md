# 집필 컨벤션

책 집필에서 흔들리면 안 되는 고정 규칙. 챕터 작성, 프로젝트 생성, 코드 레포 관리 시 모두 참조한다.

---

## 1. 챕터 표기

**`Ch01` 형식으로 통일** (두 자리 숫자 고정).

| 위치 | 형식 | 예시 |
|------|------|------|
| H1 제목 | `# Ch{NN}: 제목 (ex{NN})` | `# Ch07: 캐시와 모니터링 (ex07)` |
| 내부 참조 | `Ch{NN}에서` | "Ch06의 에이전트를 이어받아" |
| 목차 | `Ch{NN} 제목` | `Ch07 캐시와 모니터링` |
| 이론 챕터 | `# Ch{NN}: 제목` (실습 없음) | `# Ch01: RAG가 뭔가요` |

**금지**: `CH01`(대문자), `Ch.1`(점), `챕터 01`(한글), `제1장` 표기.

---

## 2. 코드 폴더명

**`ex{NN}`으로 통일**. `code/`, `rag-start/`, `rag-end/` 모두 동일.

```
code/ex01/
rag-start/ex01/
rag-end/ex01/
```

챕터 번호와 1:1 매핑. `Ch02` 실습 → `ex02`.

---

## 3. 이론/실습 챕터 싱크

| 구분 | 책 | Git 레포 |
|------|-----|----------|
| 챕터 번호 | 연속 (Ch01, Ch02, Ch03...) | 실습 챕터만 폴더 존재 |
| 이론 챕터 | 코드 없이 본문만 | 폴더 **건너뛴다** |

**예시**: 책이 `Ch01(이론) → Ch02(실습) → Ch03(실습)`이면 Git은 `ex02, ex03`만 존재.

번호는 책 기준으로 싱크 (ex02는 반드시 Ch02).

---

## 4. 챕터 마크다운 파일명

```
chapters/01-환각과-RAG의-첫-만남.md
book/body/01-환각과-RAG의-첫-만남.md
```

- 형식: `NN-한글제목.md` (두 자리 번호 + 하이픈 + 한글)
- 공백 대신 하이픈
- `Ch01-` 접두사 붙이지 않음 (폴더가 chapters이므로 중복)

## 5. 이미지 에셋 경로

```
assets/Ch{NN}/{유형}/{NN}_{id}.png
```

| 항목 | 규칙 |
|------|------|
| 챕터 폴더 | `Ch07/` (첫 글자 대문자 Ch, 두 자리 숫자) |
| 유형 서브폴더 | `gemini/`, `terminal/`, `diagram/`, `browser/` |

(코드 블록 0건)

## 7. H3 제목 패턴

### 이야기 파트 H3

```
### N.M 기술명 -- 비유/설명
```

예: `### 1.4 QueryRouter -- 3단계 질문 분류`

### 기술 파트 실습 H3

```
### N.M 실습 K: 기술명 -- 비유 (파일명)
```

예: `### 2.5 실습 1: ResponseCache + EmbeddingCache -- 사서의 메모장 두 권 (cache.py)`

기술명을 앞에 두어 독자가 목차만 봐도 무엇을 배우는지 파악할 수 있게 한다.

## 8. 이미지 플레이스홀더 4종

| 플레이스홀더 | 용도 | 처리 주체 |
|--------------|------|-----------|
| `[GEMINI PROMPT]` | 개념/비유 도식 (정적 일러스트) | 유저가 Gemini에 프롬프트 입력 |
| `[FLOW CARD]` | 단계별 흐름/파이프라인 | visual 스킬이 D2로 렌더 |
| `[CAPTURE NEEDED]` | 터미널/브라우저 실행 결과 | screenshot 스킬 |
| ```` ```mermaid ```` | 간단한 시퀀스 (인라인) | Pandoc 자동 렌더 |

**구분 기준**:
- 비유/추상 개념을 그림으로 → GEMINI PROMPT
- 단계가 순서대로 흘러가는 파이프라인 → FLOW CARD
- 실제 실행 스크린샷 → CAPTURE NEEDED
- 간단한 시퀀스(캐릭터 간 호출 등) → Mermaid 코드블록

## 9. 코드 분류 태그

파일 트리에 역할을 명시한다.

```
├── foo.py      [실습] 독자가 TODO의 pass를 지우고 직접 작성
├── bar.py      [설명] 코드블록 발췌로 보여주지만 직접 작성하지 않음
├── _baz.py     [참고] 파일명 + 한 줄 역할만 언급, 코드 공개 안 함
```

- `[실습]`: 챕터 핵심. TODO+pass 스켈레톤을 rag-start에 제공
- `[설명]`: 중요하지만 타이핑 부담 줄이려고 읽기만
- `[참고]`: 심화. 파일명이 `_`로 시작하는 경우가 많음

---

## 10. 실습 코드 레포 2개

| 레포 | 용도 | 상태 |
|------|------|------|
| `{book}-start` (예: `rag-start`) | 독자 실습용 | TODO + `pass` 스켈레톤 |
| `{book}-end` (예: `rag-end`) | 완성본 참고 | 실행 가능한 전체 코드 |

폴더 구조는 동일 (`ex01/`, `ex02/`...). 파일 경로도 동일.

독자 흐름: `start` 클론 → 챕터 따라 코드 작성 → 막히면 `end` 참고.

## 11. 용어 표기

| 대상 | 형식 | 예시 |
|------|------|------|
| 영어 용어 첫 등장 | `한국어(English)` | `토큰(Token)`, `임베딩(Embedding)` |
| 이후 등장 | 한국어만 (필요 시 영어 혼용) | "토큰이 부족하면..." |
| 볼드 (용어 소개) | `**용어(English)** 는` (닫힌 뒤 띄어쓰기) | `**세션(Session)** 이 만료되면` |
| 볼드 (순수 텍스트) | `**Docker**를` (붙여쓰기) | `**Docker** 는 컨테이너를` |

## 12. 금지 패턴

| # | 패턴 | 대안 |
|---|------|------|
| 1 | 이모지 (본문·박스 모두) | 사용 안 함 |
| 2 | 수평선 `---` | 빈 줄 또는 다음 H2/H3로 자연 전환 |
| 3 | 번호 주석 ①②③ (코드 내외 모두) | `# 1. 설명`, "1단계" |
| 4 | 같은 어미 3연속 | 어미 변주 (-합니다/-입니다/-죠/-보세요) |
| 5 | "~게 됩니다" 패턴 | "~합니다", "~입니다" |
| 6 | "이야기 파트" / "기술 파트" 메타 라벨 | 자연스러운 H2 제목 |
| 7 | 설교 ("이것이야말로", "이것이 바로...이유") | 사용 안 함 |
| 8 | 정의 전 용어 사용 | 비유로 먼저, 용어 정리에서 정식 정의 |

## 13. 캡션

- 이미지 바로 아래 `*캡션*` 이탤릭
- 존댓말 (`-합니다`, `-입니다`)
- `그림 N-N:` 접두어 붙이지 않음 (Typst가 자동 부여)
- `![](경로)`의 alt 텍스트는 비움

```markdown
![](../assets/Ch07/gemini/07_cache-concept.png)
*같은 질문이 또 오면 서가에 가지 않고 메모장을 읽어줍니다*
```

## 14. 블록쿼트 헤더

H1 직후 블록쿼트로 챕터 요약을 표시한다.

```markdown
> 한 줄 요약: (한 줄로 챕터의 본질을 표현)
> 핵심 개념: (3개 이하, 쉼표로 구분)
```

- `한 줄 요약`과 `핵심 개념` 두 줄 고정
- 각 항목은 별도 `>` 라인

## 15. 버전 관리

### 플래닝 산출물 (덮어쓰지 않음)

`-vN` 접미사로 버전 관리.

```
planning/
├── seed-v1.md
├── seed-v2.md
├── outline-v1.md
├── outline-v2.md
└── scenario-v1.md
```

### 챕터 본문 (덮어쓰기)

`chapters/01-제목.md`는 그대로 수정. Git 히스토리로 버전 추적.

### 이미지 에셋 (덮어쓰기)

동일 경로에 덮어쓴다. 레이아웃이 바뀌면 id를 바꿔서 새 파일.

## 16. 새 책 시작 시 체크리스트

1. `projects/[책이름]/` 디렉토리 구조 생성
2. `code/` 폴더 생성 (실습 챕터별 `ex{NN}/`)
3. Git 레포 2개 개설: `{book}-start`, `{book}-end`
4. 챕터 번호와 `ex{NN}` 번호 싱크 확인 (이론 챕터는 ex 없음)
5. 이 컨벤션 문서를 참조로 지정
