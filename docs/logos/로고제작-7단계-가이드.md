## 전제

로고 = 네이밍이 아니다. 브랜드를 만들려면:

1. 네이밍(브랜드의 이름)이 필요하다
2. 네이밍이 있으려면 '슬로건' 즉 브랜드의 이념이 있어야 한다
3. 브랜드의 이념이 있으려면 '타겟'을 먼저 정해야 한다

순서: **기업의 슬로건 & 이념 정하기 -> 네이밍 정하기 -> 로고 제작 -> 상품 제작**

---

## 1단계. 브랜드 아이덴티티 (BI Design)

- 슬로건을 정하면 브랜드 이념 정립이 쉬워진다
- 킬링 한 줄이 기업의 이미지를 대표하고 모든 제품/디자인의 지표가 된다
- **아이덴티티를 먼저 정하고 네이밍을 정하는 게 쉽다**
- 슬로건이 애매모호하면 안 된다 -> 확실하게

## 2단계. 디자인 디깅 (Digging)

- 디깅: 만들기 전 그 분야의 자료조사. 비슷한 업종의 디자인 스타일, 서체, 색감 조사
- **동종 업계 자료조사 필수**
- 타겟층이 좋아하는 스타일의 디자인 자료조사
- 목적: 1) 트렌드 파악 2) 디자인 스타일 선택 3) 디자인 퀄리티 파악
- 주의: 따라하면 안 된다. 독보적인 브랜드가 되어야 한다

## 3단계. 디자인 스타일 & 색감 고르기

### 서체 활용
- **세리프**: 세련되고 무드 있는 브랜드
- **산세리프**: 세련되고 깔끔한 브랜드
- **캘리그라피**: 자유롭고 통통 튀는 브랜드

### 형태 활용
- 도형, 모노그램, 엠블럼, 캐릭터 등

### 색감 활용
- 색마다 주는 느낌이 다르니 브랜드와 어울리는 컬러 조합 선택
- 로고 가이드에 컬러 베리에이션(변형) 다양한 조합 포함
- 인쇄 컬러칩(팬톤) 참고 추천

## 4단계. 서체 활용방법

| 서체 유형 | 특징 | 어울리는 브랜드 |
|-----------|------|----------------|
| 세리프 (명조) | 세련, 무드 | 프리미엄, 전문적 |
| 산세리프 (고딕) | 깔끔, 현대적 | 테크, 모던 |
| 캘리그라피 | 자유, 개성 | 아트, 크리에이티브 |
| 서체 제작 | 고유 아이덴티티 | 시그니처 브랜드 |

- 기존 폰트 변형 시 저작권 확인 (로고 포함 여부, 변형 허용 여부)
- 어중간하게 만들 바엔 라이선스 부합 폰트 구매/사용

## 5단계. 형태 활용방법

| 형태 | 설명 | 적합한 경우 |
|------|------|------------|
| (5-1) 도형화 로고 | 의미를 도형으로 표현 | 심플, 현대적 |
| (5-2) 엠블럼 로고 | 전통 깊은 느낌 | 빈티지, 레트로 |
| (5-3) 모노그램 로고 | 글자 2개+ 조합 기호 | 이니셜, 세련 |
| (5-4) 캐릭터 로고 | 캐릭터 활용 | 제품 다양화 |

- 형태는 로고 무드에 맞게 선택 (빈티지->엠블럼, 세련->이니셜)

## 6단계. 스케치 후 제작

- **반드시 스케치를 하고 작업** -> 다양한 형태 확인 후 진행
- 스케치 후 **일러스트레이터** 프로그램 사용 (포토샵 금지)
- 벡터 형식: 확대/축소 시 화질 유지, 변형 용이

## 7단계. 로고 활용하여 브랜딩 디자인 작업하기

- 로고 플레이는 적당히
- 잘 만든 로고는 타입을 다양하게 변형 가능
- 캐릭터 있으면 제품 활용도 높음
- 캐릭터 없으면 패턴이나 로고를 패턴처럼 활용

## OpenSkillBooks 적용 계획

### 1단계 결과: 브랜드 아이덴티티
- **슬로건**: "따라하면 만들어지는 IT 기술서"
- **이념**: 간결한 코드와 일상 비유로 누구나 따라할 수 있는 실습형 기술서
- **타겟**: 개발자, IT 학습자 (배경지식은 있지만 특정 기술만 모르는 사람)
- **네이밍**: OpenSkillBooks (오픈스킬북스) — "열린 기술 책"

### 2단계 결과: 디깅 (경쟁사 분석)
- O'Reilly: 동물 일러스트 + 세리프 (전통, 신뢰)
- Manning: 역사 인물 + 세리프 (격식)
- Packt: 산세리프 + 기하학 (모던)
- 한빛미디어: 산세리프 + 깔끔 (실용)
- 위키북스: 세리프 + 미니멀 (전문)
- **차별점**: 코드 + 책의 융합, 개발자 친화적 아이콘

### 3단계 결과: 스타일 & 색감
- **서체**: 산세리프 (모던/테크) + 모노스페이스 (코드 느낌)
- **형태**: 도형화 (코드 페이지, 펼친 책, 브래킷 프레임)
- **색감**: 4가지 팔레트 (Black+Teal, Navy+Sky, Teal+Amber, Mono)

### 4~5단계: 서체 + 형태 (12 조합)
- 이미 v4에서 3가지 아이콘 x 4가지 스타일로 구현 완료

### 6단계: 스케치 -> 제작
- SVG 벡터 형식으로 제작 (확대/축소 무한, 변형 용이)
- Python 스크립트로 48종 자동 생성

### 7단계: 브랜딩 활용
- 책 표지, 웹사이트, 명함, SNS 프로필에 적용
- 컬러 베리에이션 4종으로 다양한 배경에 대응
