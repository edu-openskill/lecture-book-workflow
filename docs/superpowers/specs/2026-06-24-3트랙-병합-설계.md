# 3-트랙 집필 하네스 병합 설계

날짜: 2026-06-24
대상 저장소: `book-workflow/` (현재 폴더의 git 저장소)
베이스 브랜치: `feature/개념트랙-이미지자동화` (현재) → 작업은 새 브랜치에서

---

## 1. 목표

책 집필 하네스를 **3개 트랙으로 분기**시키고, 하네스를 시작하면 사용자가 셋 중 하나를 고르게 한다.

| 트랙 | 사용자 분류 | 한 줄 정의 |
|------|------------|-----------|
| `lecture` | #1 유튜브 영상으로 책 만들기 | 녹화된 강의(판서대본 + 유튜브 자막)를 원고 소스로 |
| `code` | #2 소스코드 기반으로 책 만들기 | 완성 코드를 의도 필터로 분석해 책으로 |
| `concept` | #3 목차를 기준으로 책 만들기 | 완성물 없이 목차/주제만으로 (맨바닥 집필) |

`lecture`는 별도 저장소 `github.com/edu-openskill/lecture-book-workflow`에 존재한다.
이 저장소를 현재 `book-workflow`에 병합하여, 이미 존재하는 `code`/`concept` 2-트랙 분기를 3-트랙으로 확장한다.

---

## 2. 현황 (병합 전)

### 현재 `book-workflow` — 이미 2-트랙 구조

CLAUDE.md에 `track` 변수(`code` 기본 / `concept`)가 이미 존재하며, `새 책 만들기` 시 트랙을 묻고 STEP 2·3·4에서 분기한다.

- 트랙 전용 폴더: `code` → `code/` + `versions/`, `concept` → `examples/`
- 고유 스킬: `code`, `math`, `image-gen`, `pdf`, `pdf-ty`
- 공유 스킬: `writing`, `review`, `planning`, `visual`, `screenshot`, `pub-*`, `humanizer`, `design-doc-mermaid`

### `lecture-book-workflow` — book-workflow 포크

입력 소스를 "판서대본 + 유튜브 자막"으로, STEP 2를 강의 해부로 개조한 버전.

- 고유 스킬: `lecture`(강의 해부), `lecture-caption`(yt-dlp .vtt 다운로드·정제·정렬)
- 고유 워크플로우: `workflow/step2-강의해부.md`, `workflow/step3-챕터매핑.md`
- 제거됨: `code`/`math`/`image-gen`/`pdf` 스킬군
- 입력 폴더: `sources/NN강/{판서대본.md, caption-raw.vtt, caption-clean.md, 정렬표.md}`

### 두 저장소 차이 요약

| | 현재 book-workflow | lecture-book-workflow |
|---|---|---|
| 입력 | `code/` 또는 `examples/` | `sources/NN강/` |
| STEP 2 | 코드해부 / 개념분석 | 강의해부 |
| STEP 3 | 시나리오+버전 / 시나리오 | 챕터매핑 |
| 고유 스킬 | code, math, image-gen, pdf | lecture, lecture-caption |

## 3. 채택 접근법 — A: 단일 repo + `track` 3값 확장

현재 `book-workflow`를 베이스로 두고, `track` 변수를 `{lecture, code, concept}` 3값으로 확장한다.
lecture 능력(스킬 2개 + step2·3 lecture 버전)을 **추가만** 하고, 어떤 것도 제거하지 않는다.
공유 스킬은 1벌만 유지한다.

**기각한 대안:**
- **B (3폴더 + 런처)**: 공유 스킬이 2~3벌 중복, 동기화 부담. 단일 `track` 설계 철학과 충돌.
- **C (workflow를 트랙별 하위폴더로 재편)**: 더 깔끔하나 기존 경로를 전부 바꾸는 큰 리팩토링. A 완료 후 선택적으로 부분 도입 가능(필수 아님).

---

## 4. 트랙 모델

하나의 변수 `track`, 세 값. STEP 1·4·5·6·7 + 검토모드 3개 + 문체상수는 **세 트랙 공유**. 분기는 STEP 2·3과 입력 폴더, 일부 트랙 종속 상수에만 발생한다.

| track | 입력 소스 | STEP 2 | STEP 3 | 전용 폴더 | 성장 방식 |
|-------|----------|--------|--------|----------|----------|
| `lecture` | 판서대본 + 자막(.vtt) | 강의해부 | 챕터매핑 | `sources/NN강/` | 개념 누적 (한 강 = 한 장) |
| `code` | 완성 코드 | 코드해부 | 시나리오+버전 | `code/` + `versions/` | 버전별 성장 (챕터 = 한 버전) |
| `concept` | 목차/주제 | 개념분석 | 시나리오 | `examples/` | 의존성 누적 |

## 5. 진입 흐름 (시작하자마자 분기)

```
하네스 시작 / `새 책 만들기`
  └─ STEP 0 (신규): 트랙 선택을 가장 먼저 묻는다
        "어떤 책을 쓸까요?
         1. 유튜브 강의로   2. 소스코드로   3. 목차/주제로"
       → track 확정 → progress.json.track 기록
  └─ 책 이름(프로젝트 폴더명) 입력
  └─ 트랙별 디렉토리 생성
        lecture → sources/ (+ 공통 폴더), code → code/ + versions/, concept → examples/
  └─ 트랙별 소스 입력 안내
        lecture → 판서대본 위치 + 유튜브 URL(자막은 `자막 추출`)
        code    → code/ 폴더 또는 GitHub URL
        concept → 목차/주제
  └─ 소스 사전 스캔 → STEP 1 씨앗 심기 → STEP 2·3은 track 따라 분기
```

## 6. 병합 작업 명세

### 6.1 추가 (lecture-repo → 현재 repo, 신규 파일 — 충돌 없음)

| 항목 | 출처 → 대상 |
|------|-----------|
| 강의 해부 스킬 | `lecture-repo/.claude/skills/lecture/` → `book-workflow/.claude/skills/lecture/` |
| 자막 스킬 | `lecture-repo/.claude/skills/lecture-caption/` → `book-workflow/.claude/skills/lecture-caption/` |
| STEP 2 가이드 | `lecture-repo/workflow/step2-강의해부.md` → `book-workflow/workflow/` |
| STEP 3 가이드 | `lecture-repo/workflow/step3-챕터매핑.md` → `book-workflow/workflow/` |

> `lecture-caption`은 yt-dlp 의존. 스크립트째로 가져온다(실행 환경 의존성은 스킬 내부 안내 따름).

### 6.2 수정 (공유 파일에 lecture 분기 추가 — 내용 병합)

| 파일 | 변경 |
|------|------|
| `CLAUDE.md` | (1) STEP 0 "제일 먼저 트랙 선택" 블록 추가 (2) `track` 표를 3값으로 확장 (3) 트랙 종속 상수표에 `lecture` 열 추가 (4) `새 책 만들기` 디렉토리 생성/소스 안내를 3-way 분기로 (5) `자막 추출` 명령어 추가 (6) 명령어 라우팅에 lecture STEP 2·3 연결 |
| `workflow/step1-씨앗.md` | lecture 소스(대본·자막) 사전 스캔 분기 한 줄 추가 |
| `workflow/step4-뼈대.md` | lecture 자료 배치(비유→정의) 분기 추가 |
| `workflow/step5-챕터집필.md` | lecture 챕터 원료(판서대본+자막+시뮬 캡처) 분기 추가 |
| `workflow/review-guide.md` | (필요 시) lecture 트랙 의도감시 항목 보정 |
| `.claude/progress-template.json` | `track` enum에 `lecture` 추가 |
| `.claude/skills/CATALOG.md` | `lecture`, `lecture-caption` 스킬 2개 등재 |
| `README.md` | 3-트랙 소개로 갱신 |

### 6.3 유지 (현재 repo 고유 — 손대지 않음)

`code`, `math`, `image-gen`, `pdf`, `pdf-ty` 스킬과 `step2-코드해부.md`, `step3-시나리오.md`는 각 트랙 소유로 그대로 둔다.

### 6.4 가져오지 않음

`lecture-repo/projects/` 의 샘플 프로젝트 산출물. 스킬·워크플로우·폴더 규약만 병합한다.

---

## 7. 안전장치 / 제약

1. **새 git 브랜치에서 작업** (예: `feature/3-track-merge`). 현재 `feature/개념트랙-이미지자동화`는 건드리지 않는다.
2. **추가만, 삭제 없음.** 현재 repo의 어떤 트랙 능력도 제거하지 않는다.
3. **파일명 충돌 0.** 신규 추가 파일(lecture 스킬·step2-강의해부·step3-챕터매핑)은 현재 repo와 겹치지 않는다. 겹치는 step1/4/5/review-guide/CLAUDE.md는 내용 병합으로 처리.
4. lecture-repo는 scratchpad에 clone 되어 있다(원본 보존, 직접 수정 안 함).
