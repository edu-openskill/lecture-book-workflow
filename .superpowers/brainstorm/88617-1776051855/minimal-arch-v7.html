ConnectHR 아키텍처 — 미니멀 v7 (흐름 번호)

각 흐름에 1~7 번호 표시 — 쿼리가 처리되는 순서를 한눈에

쿼리 처리 흐름
  1. 사내 직원 → FastAPI: 사용자가 사내 대시보드에서 질문
  2. FastAPI → Query Router: API가 질문을 오케스트레이션 계층으로 전달
  3. Query Router → ReAct Agent: 질문 유형 분류 (DB 조회? 문서 검색?)
  4. ReAct Agent → MCP Tools / LCEL Chain: 결정된 경로로 정보 수집 시작
    • 4a. DB 경로 → MCP Tools
    • 4b. RAG 경로 → LCEL Chain (튜닝 레이어 선택적 거침)
  5. MCP Tools → PostgreSQL / LCEL Chain → ChromaDB: 실제 데이터 읽기 (파랑)
  6. ReAct Agent → Ollama LLM: 수집된 컨텍스트로 LLM 추론 (외부)
  7. ReAct Agent → 사내 직원: 응답 반환 (파랑, 좌측 복도)

별도 흐름 I: Doc Pipeline → ChromaDB (문서 인제스천 · 오프라인, 최초 1회 + 재인덱싱 시). Ch.3 규칙이 이 파이프라인을 지배.
