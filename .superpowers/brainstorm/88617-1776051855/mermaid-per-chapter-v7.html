# 단계 입력 출력 Ch
1 질문 접수 — 사내 직원이 대시보드에서 자연어 질문 자연어 텍스트 HTTP POST 요청 Ch.2
2 라우팅 — FastAPI가 /chat 엔드포인트로 수신해 오케스트레이션 계층 호출 JSON {query, session_id} 라우터 입력 Ch.2·6
3 분류 — Query Router가 규칙·스키마·LLM 3단계로 질문 타입 결정 원본 질문 경로 태그 (DB/RAG) Ch.6
4 도구 호출 — Agent Chain이 ReAct 패턴으로 MCP Tools 또는 LCEL Chain 선택 경로 + 질문 Tool call 명령 Ch.6
5 데이터 조회 — MCP→PostgreSQL (SQL) 또는 LCEL→ChromaDB (벡터 검색) SQL / 임베딩 벡터 행 / 관련 문서 k개 Ch.2·4
6 LLM 호출 — Agent가 수집한 컨텍스트를 프롬프트로 만들어 Ollama 호출 컨텍스트 + 프롬프트 LLM 입력 토큰 Ch.5·6
7 LLM 답변 — DeepSeek-R1:8b가 Chain-of-Thought으로 답변 생성 (stream) 프롬프트 답변 텍스트 + 출처 Ch.1
8 응답 반환 — Agent가 답변·출처 정리 후 FastAPI를 거쳐 JSON으로 사용자에게 LLM 원본 답변 JSON {answer, sources, tokens} Ch.2·6
부가 흐름: • 오프라인 파싱·벡터화: Ch.3 규칙에 맞춰 준비된 문서 → Doc Pipeline → ChromaDB (재인덱싱 주기적) • 운영 감시: 모든 요청·응답을 Cache/TokenTracker가 기록 (Ch.7)
