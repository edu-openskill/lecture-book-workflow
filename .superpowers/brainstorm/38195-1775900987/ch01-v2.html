Ch.1 AI한테 물어봤더니 거짓말을 합니다

환각, 컨텍스트 주입, RAG의 기본 원리

이번 챕터가 끝나면
  • LLM은 모르는 것을 자신 있게 지어낸다는 걸 직접 확인합니다
  • 문서를 건네주면 거짓말을 멈추는 이유를 이해합니다
  • 이걸 자동화한 RAG 파이프라인을 직접 만들어봅니다

1.1 그럴듯한 거짓말

입사 3일 차.

오픈이는 모니터 앞에서 턱을 괴고 있었습니다. 인사팀에서 넘어온 메일이 화면을 채우고 있었습니다. "직원들이 회사 규정을 쉽게 물어볼 수 있는 AI 비서, 프로토타입 가능할까요?" 금요일까지. 이틀 남았습니다.

팀장이 커피를 들고 다가왔습니다.

팀장: "어제 ChatGPT한테 우리 회사 연차 규정 물어봤거든. 그럴듯하게 대답하더라. 근데 틀렸어."

*ChatGPT도 뭐든 대답하잖아. 그냥 물어보면 되는 거 아니야?*

정말 틀리는지 직접 확인해보겠습니다. Ollama라는 도구를 쓰면 내 컴퓨터에서 LLM을 돌릴 수 있어요. deepseek-r1이라는 8B 파라미터짜리 모델을 올려두었습니다.

실습 step1_fail.py — LLM 단독 호출로 환각 체험
from langchain_ollama import ChatOllama from rich.console import Console console = Console() # 1. Ollama에서 deepseek-r1:8b 모델을 로드 llm = ChatOllama(model="deepseek-r1:8b", temperature=0) question = "우리 회사(커넥트)의 신입사원 연차 발생 규정이 어떻게 돼?" # 2. 질문 출력 → LLM 호출 → 답변 출력 console.print(f"[bold]질문:[/bold] {question}\n") response = llm.invoke(question) console.print(f"[bold]답변:[/bold]\n{response.content}")

temperature=0은 "가장 확률 높은 답변만 내놓아라"라는 설정입니다. 창의성을 낮추면 정확해지지 않을까 기대하면서 엔터를 눌렀습니다.

실행 결과 — 환각
질문: 우리 회사(커넥트)의 신입사원 연차 발생 규정이 어떻게 돼?

답변: 커넥트 사의 신입사원 연차 발생 규정은 근로기준법 제60조에 따라 입사 후 1년간 매월 1일의 유급휴가가 발생하며, 1년 근속 시 15일의 연차가 부여됩니다...

조항 번호까지 달려 있으니 누구라도 믿을 만합니다. 그런데 오픈이가 인사팀에서 받은 실제 규정집을 열어보니, 첫 줄부터 달랐습니다. 커넥트의 실제 규정은 "신입사원은 입사 후 3년간 연차 없음, 대신 매월 리프레시 데이 제공"이었습니다.

*조항 번호까지 붙여놓고 전부 지어낸 거야?*

LLM의 답변 vs 실제 규정
LLM의 답변 (환각)
근로기준법 제60조 적용

이 현상에는 이름이 있습니다. 환각(Hallucination).

유령의 집에 들어가면 귀신이 진짜처럼 보이잖아요? 하지만 조명이 켜지면 전부 마네킹이고 세트입니다. AI의 답변도 같습니다. 형식은 완벽한데 내용은 허구예요. 이 모델은 커넥트라는 회사를 학습한 적이 없습니다. 없는 정보를 요청받으면 "모릅니다" 대신 있을 법한 답을 지어냅니다.

[GEMINI PROMPT] 환각 개념도
LLM이 환각을 일으키는 과정을 시각화한 개념도
프롬프트: 교육용 개념도를 그려주세요. 왼쪽에 사용자가 "우리 회사 연차 규정 알려줘"라고 질문합니다. 가운데에 LLM(뇌 모양)이 있고, LLM 내부에는 "학습 데이터: 일반 법률, 뉴스, 위키백과"가 적혀 있습니다. "커넥트 규정"은 학습 데이터에 없다는 표시(X 또는 빈칸). 오른쪽에 LLM이 "근로기준법 제60조에 따라..."라고 자신 있게 답변하는데, 빨간색으로 "지어낸 답변"이라고 표시. 한국어, 깔끔한 플랫 디자인, 흰 배경.

1.2 교재를 펼쳐놓으면

그러면 어떻게 해야 할까요? 팀장이 힌트를 줬습니다.

팀장: "전부 외우게 하지 말고, 필요한 문서만 찾아서 읽게 해."

오픈북 시험이 떠올랐습니다. 시험 범위를 전부 외우는 대신, 교재를 옆에 펼쳐놓고 필요한 부분을 찾아서 답을 쓰는 방식. AI한테도 교재를 직접 건네주면 되지 않을까요?

커넥트의 실제 취업규칙을 변수에 담아서 프롬프트에 넣어봤습니다.

실습 step2_context.py — 컨텍스트 직접 주입
from langchain_ollama import ChatOllama from rich.console import Console console = Console() llm = ChatOllama(model="deepseek-r1:8b", temperature=0) # 1. 실제 사내 규정을 변수에 담는다 context_data = """ [커넥트 취업규칙] 1. 신입사원은 입사 후 3년 동안은 연차가 없다. 2. 대신 매월 1회 '리프레시 데이'를 유급으로 제공한다. 3. 3년 근속 시 30일의 연차가 일시에 발생한다. """ question = "우리 회사(커넥트)의 신입사원 연차 발생 규정이 어떻게 돼?" # 2. 규정 문서를 프롬프트에 직접 넣어서 전달 prompt = f""" 아래 [참고 정보]를 보고 질문에 답해줘. [참고 정보] {context_data} 질문: {question} """ console.print(f"[bold]질문:[/bold] {question}\n") response = llm.invoke(prompt) console.print(f"[bold]답변:[/bold]\n{response.content}")
실행 결과 — 정확
질문: 우리 회사(커넥트)의 신입사원 연차 발생 규정이 어떻게 돼?

답변: 커넥트의 신입사원은 입사 후 3년 동안은 법정 연차가 발생하지 않습니다. 대신 매월 1회 유급 '리프레시 데이'가 제공됩니다. 3년 근속 시 30일의 연차가 일시에 부여됩니다.

규정집과 한 글자도 다르지 않았습니다.

*문서를 넘겨주니까 거짓말을 안 하네.*

이걸 컨텍스트 주입(Context Injection)이라고 합니다. 시험지 옆에 교재를 펼쳐놓는 것과 같은 원리예요. AI는 주어진 문서 안에서만 답을 찾으니까, 지어낼 이유가 없어집니다.

1.3 사서가 필요합니다

그런데 기쁨은 오래가지 않았습니다.

지금은 규정 하나를 직접 복사해서 넣었잖아요. 커넥트의 사내 문서가 인사규정, 보안규정, 복지규정, 출장규정... 수십 개입니다. 직원이 "보안 USB 규정이 뭐야?"라고 물으면 보안규정을 찾아서 넣어줘야 하고, "야근 식대 되나요?"라고 물으면 복지규정을 찾아서 넣어줘야 합니다.

동료: "그걸 사람이 해주면 AI가 왜 필요해?"

맞는 말이었습니다. 이걸 자동으로 해줘야 합니다.

도서관을 떠올려보겠습니다. 서가에 책이 꽂혀 있고, 누군가 질문하면 사서가 관련 책을 찾아서 건네줍니다. 지금 필요한 게 바로 그 사서입니다. 질문이 들어오면 수십 개의 문서 중에서 관련된 것만 자동으로 골라서 LLM에 넘겨주는 구조.

이 구조의 이름이 RAG(Retrieval-Augmented Generation)입니다. 직역하면 "검색으로 강화한 생성". 쉽게 말하면, 도서관에 사서를 앉혀놓는 겁니다.

RAG 파이프라인 — 사서가 일하는 순서
📚
1
문서 저장
문서를 숫자 벡터로 변환해서
벡터DB에 저장
(서가에 책 꽂기)
➜
🔎
2
문서 검색
질문과 가장 비슷한
문서를 자동으로 찾기
(사서가 책 찾기)
➜
💬
3
답변 생성
찾은 문서를 LLM에
넘겨서 답변 생성
(AI가 읽고 답하기)

1.3.1 서가에 책 꽂기 — 문서 준비와 임베딩

사내 규정 3개를 문서로 만들어서 RAG 파이프라인을 구성해보겠습니다. 먼저 문서를 준비하고, 이 문서들을 숫자 벡터로 바꿔서 검색 가능한 상태로 만듭니다.

실습 step3_rag.py — RAG 파이프라인 구성 (1/2: 문서 저장)
from langchain_classic.chains import RetrievalQA from langchain_chroma import Chroma from langchain_ollama import OllamaEmbeddings, ChatOllama from langchain_core.documents import Document from langchain_core.prompts import PromptTemplate # 1. 사내 규정 3개를 문서로 준비 docs = [ Document( page_content="[인사규정] 신입사원은 입사 후 3년간 연차 없음. 매월 1회 유급 리프레시 데이 제공.", metadata={"source": "인사규정"}), Document( page_content="[보안규정] 회사 지급 승인 보안 USB만 사용. 개인 USB 사용 금지.", metadata={"source": "보안규정"}), Document( page_content="[복지규정] 점심은 무제한 법인카드. 저녁은 오후 9시 이후 야근 시만 가능.", metadata={"source": "복지규정"}), ] # 2. 문서를 벡터로 변환해서 ChromaDB에 저장 embeddings = OllamaEmbeddings(model="nomic-embed-text") vectorstore = Chroma.from_documents(documents=docs, embedding=embeddings) # 3. 질문과 가장 비슷한 문서를 찾는 검색기 retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

nomic-embed-text는 문서의 의미를 숫자 벡터로 변환하는 임베딩 모델입니다. ChromaDB는 이 벡터들을 저장하고 검색해주는 벡터 데이터베이스예요. 사서에게 서가를 준 겁니다.

1.3.2 사서에게 질문하기 — 검색 + 답변

이제 검색기와 LLM을 하나의 체인으로 연결합니다.

실습 step3_rag.py — RAG 파이프라인 구성 (2/2: 체인 연결)
# 4. 프롬프트 템플릿 template = """당신은 회사의 규정에 대해 설명해주는 AI 비서입니다. 아래의 참고 정보를 바탕으로 질문에 답하세요. 참고 정보: {context} 질문: {question} 답변:""" PROMPT = PromptTemplate(template=template, input_variables=["context", "question"]) # 5. 검색기 + LLM을 체인으로 연결 llm = ChatOllama(model="deepseek-r1:8b", temperature=0) qa_chain = RetrievalQA.from_chain_type( llm=llm, retriever=retriever, return_source_documents=True, chain_type_kwargs={"prompt": PROMPT} ) # 6. 질문 실행 result = qa_chain.invoke({"query": "신입사원 휴가 규정에 대해 알려줘."})

RetrievalQA가 전체 과정을 자동으로 처리합니다. 질문이 들어오면 retriever가 관련 문서를 찾고, 찾은 문서를 프롬프트에 넣어서 LLM에 보내고, 답변을 받아옵니다. step2에서 우리가 손으로 했던 일을 코드가 대신하는 겁니다.

실행 결과
--- 검색된 문서(근거) ---
[인사규정]: 신입사원은 입사 후 3년간 연차 없음. 매월 1회 유급 리프레시 데이 제공.
[복지규정]: 점심은 무제한 법인카드. 저녁은 오후 9시 이후 야근 시만 가능.
[보안규정]: 회사 지급 승인 보안 USB만 사용...

--- AI 답변 ---
신입사원은 입사 후 처음 3년 동안은 법정 연차가 발생하지 않습니다. 대신 매월 1회의 유급 '리프레시 데이'를 휴가로 사용할 수 있습니다.

사서가 일을 했습니다. "신입사원 휴가"라는 질문에 서가에서 인사규정을 찾아서 AI에게 건넸고, AI는 그 문서를 읽고 정확하게 답했습니다. 검색된 근거까지 함께 보여주니 어떤 문서를 참고했는지도 확인할 수 있습니다.

[GEMINI PROMPT] step1(환각) vs step3(RAG) 비교 흐름도
같은 질문에 대해 step1(LLM 단독)과 step3(RAG)의 처리 과정과 결과를 나란히 비교하는 흐름도
프롬프트: 교육용 비교 흐름도를 그려주세요. 위쪽은 "LLM 단독 (step1)" — 사용자 질문 → LLM(학습 데이터에 없음) → 빨간색 "환각 답변: 근로기준법 60조...". 아래쪽은 "RAG (step3)" — 사용자 질문 → 벡터DB에서 인사규정 검색 → LLM + 검색된 문서 → 초록색 "정확한 답변: 3년간 연차 없음, 리프레시 데이...". 두 흐름의 차이점(벡터DB 검색 단계)을 강조. 한국어, 플랫 디자인, 흰 배경.

이것만은 기억하자

LLM은 모르는 것을 자신 있게 지어냅니다. 이걸 환각이라고 합니다.

문서를 직접 건네주면 해결되지만, 매번 사람이 찾아줄 수는 없습니다. RAG는 이 과정을 자동화한 구조입니다. 질문이 들어오면 관련 문서를 자동으로 찾아서 LLM에 넘겨줍니다. 도서관에 사서를 앉힌 겁니다.

다음 챕터에서는 이 사서가 답변할 데이터가 있는 사내 시스템을 먼저 살펴봅니다.
