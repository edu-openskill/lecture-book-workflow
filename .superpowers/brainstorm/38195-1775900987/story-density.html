스토리 농도 비교 — 기술과 이야기의 배합

같은 기술(환각 → 컨텍스트 주입)을 스토리 농도별로 작성했습니다. 어느 정도가 적당한지 골라주세요.

A

라이트 — 상황만 깔고 바로 기술 (스토리 20%)

도입 장면 2~3줄로 상황 설정 → 바로 코드와 설명. 기술 정보 밀도 높음

팀장이 ChatGPT에 우리 회사 연차 규정을 물어봤더니 그럴듯하게 거짓말을 했다고 합니다. 정말 그런지 직접 확인해보겠습니다. 로컬 LLM을 연결합니다. Ollama로 deepseek-r1:8b 모델을 올려두었습니다.
# step1_fail.py — TODO: LLM 단독 호출로 환각 체험 from langchain_ollama import ChatOllama llm = ChatOllama(model="deepseek-r1:8b", temperature=0) question = "우리 회사(커넥트)의 신입사원 연차 발생 규정이 어떻게 돼?" response = llm.invoke(question) print(response.content)
temperature=0은 가장 확률 높은 답변만 출력하라는 뜻입니다. 창의성을 낮추고 정확성을 높이는 설정이지만, 결과를 보면 여전히 틀립니다. 실제 규정집과 첫 줄부터 다릅니다. 이 현상을 환각(Hallucination)이라고 합니다. LLM이 학습하지 않은 정보를 그럴듯하게 지어내는 겁니다. 해결 방법은 간단합니다. 문서를 직접 넘겨주면 됩니다.

이번에는 규정집과 정확히 일치합니다. 문서를 주니까 거짓말을 안 합니다. 하지만 사내 문서가 수백 개라면? 매번 수동으로 찾아서 넣을 수는 없습니다. 이걸 자동화하는 구조가 RAG입니다.
B

미디엄 — 이야기와 코드가 교차 (스토리 50%)

캐릭터가 문제를 만나고, 코드를 짜고, 결과에 반응하는 흐름. 감정과 기술이 번갈아 등장

입사 3일 차, 오픈이에게 첫 미션이 떨어졌습니다. 사내 AI 비서를 만들라는 겁니다. 팀장이 커피를 들고 다가왔습니다. 팀장: "ChatGPT한테 우리 회사 연차 규정 물어봤더니, 그럴듯하게 거짓말을 하더라." *거짓말? AI가?* 정말 그런지 직접 확인해볼까요? 로컬에서 돌릴 수 있는 LLM을 하나 연결해보겠습니다. Ollama라는 도구에 deepseek-r1 모델을 올려두었습니다.

AI도 마찬가지로, 형식은 완벽한데 내용은 허구입니다. 팀장이 힌트를 던졌습니다. 팀장: "전부 외우게 하지 말고, 필요한 문서만 찾아서 읽게 해." 오픈북 시험이 떠올랐습니다. 시험 범위를 전부 외우는 대신, 교재를 펼쳐놓고 답을 찾는 방식. AI한테도 교재를 건네주면?

C

리치 — 이야기 안에 코드가 녹아듦 (스토리 70%)

소설을 읽다가 자연스럽게 코드를 만나는 느낌. 감정곡선/장면묘사가 풍부

"AI 비서 프로토타입, 이번 주 금요일까지 가능할까요?" 금요일이면 이틀 남았습니다. 뭘 어디서부터 해야 하는지도 모르는데. 팀장이 커피를 들고 다가왔습니다. 잠깐 앉더니 대뜸 물었습니다. 팀장: "어제 ChatGPT한테 우리 회사 연차 규정 물어봐봤거든."
