Ch.1 AI한테 물어봤더니 거짓말을 합니다

환각, 컨텍스트 주입, RAG의 기본 원리

이번 챕터가 끝나면
  • LLM은 모르는 것을 자신 있게 지어낸다는 걸 직접 확인합니다
  • 문서를 건네주면 거짓말을 멈추는 이유를 이해합니다
  • 이걸 자동화한 구조, RAG 파이프라인을 직접 만들어봅니다

입사 3일 차.

오픈이는 모니터 앞에서 턱을 괴고 있었습니다. 인사팀에서 넘어온 메일이 화면을 채우고 있었습니다. "직원들이 회사 규정을 쉽게 물어볼 수 있는 AI 비서, 프로토타입 가능할까요?"

금요일까지. 이틀 남았습니다.

팀장이 커피를 들고 다가왔습니다.

팀장: "어제 ChatGPT한테 우리 회사 연차 규정 물어봤거든. 그럴듯하게 대답하더라. 근데 틀렸어."

*ChatGPT도 뭐든 대답하잖아. 그냥 물어보면 되는 거 아니야?*

정말 틀리는지 직접 확인해보겠습니다. Ollama라는 도구를 쓰면 내 컴퓨터에서 LLM을 돌릴 수 있어요. deepseek-r1이라는 8B 파라미터짜리 모델을 올려두었습니다.

질문: 우리 회사(커넥트)의 신입사원 연차 발생 규정이 어떻게 돼?

답변: 커넥트 사의 신입사원 연차 발생 규정은 근로기준법 제60조에 따라 입사 후 1년간 매월 1일의 유급휴가가 발생하며, 1년 근속 시 15일의 연차가 부여됩니다...

조항 번호까지 달려 있으니 누구라도 믿을 만합니다.

오픈이는 인사팀에서 받은 실제 규정집을 열어봤습니다. 첫 줄부터 달랐습니다. 커넥트의 실제 규정은 "신입사원은 입사 후 3년간 연차 없음, 대신 매월 리프레시 데이 제공"이었습니다. 근로기준법 제60조와는 완전히 다른, 커넥트만의 파격적인 규정이었습니다.

*조항 번호까지 붙여놓고 전부 지어낸 거야?*

이 현상에는 이름이 있습니다. 환각(Hallucination).

유령의 집에 들어가면 귀신이 진짜처럼 보이잖아요? 하지만 조명이 켜지면 전부 마네킹이고 세트입니다. AI의 답변도 같습니다. 형식은 완벽한데 내용은 허구예요. 이 모델은 커넥트라는 회사를 학습한 적이 없습니다. 없는 정보를 요청받으면 "모릅니다" 대신 있을 법한 답을 지어냅니다.

그러면 어떻게 해야 할까요?

팀장이 힌트를 줬습니다.

팀장: "전부 외우게 하지 말고, 필요한 문서만 찾아서 읽게 해."

오픈북 시험이 떠올랐습니다. 시험 범위를 전부 외우는 대신, 교재를 옆에 펼쳐놓고 필요한 부분을 찾아서 답을 쓰는 방식. AI한테도 교재를 직접 건네주면 되지 않을까요?

커넥트의 실제 취업규칙을 변수에 담아서 프롬프트에 넣어봤습니다.

# step2_context.py — TODO: 컨텍스트 직접 주입 from langchain_ollama import ChatOllama from rich.console import Console console = Console() llm = ChatOllama(model="deepseek-r1:8b", temperature=0) # 1. 실제 사내 규정을 변수에 담는다 context_data = """ [커넥트 취업규칙] 1. 신입사원은 입사 후 3년 동안은 연차가 없다. (파격적인 규정) 2. 대신 매월 1회 '리프레시 데이'를 유급으로 제공한다. 3. 3년 근속 시 30일의 연차가 일시에 발생한다. """ question = "우리 회사(커넥트)의 신입사원 연차 발생 규정이 어떻게 돼?" # 2. 규정 문서를 프롬프트에 직접 넣어서 전달 prompt = f""" 아래 [참고 정보]를 보고 질문에 답해줘. [참고 정보] {context_data} 질문: {question} """ console.print(f"[bold]질문:[/bold] {question}\n") response = llm.invoke(prompt) console.print(f"[bold]답변:[/bold]\n{response.content}")
실행 결과
질문: 우리 회사(커넥트)의 신입사원 연차 발생 규정이 어떻게 돼?

답변: 커넥트의 신입사원은 입사 후 3년 동안은 법정 연차가 발생하지 않습니다. 대신 매월 1회 유급 '리프레시 데이'가 제공됩니다. 3년 근속 시 30일의 연차가 일시에 부여됩니다.

규정집과 한 글자도 다르지 않았습니다.

*문서를 넘겨주니까 거짓말을 안 하네.*

됐습니다. 방향이 보이기 시작했습니다. 문서를 직접 건네주면 AI는 그 안에서만 답을 찾습니다. 이걸 컨텍스트 주입(Context Injection)이라고 합니다. 시험지 옆에 교재를 펼쳐놓는 것과 같은 원리예요.

그런데 기쁨은 오래가지 않았습니다.

지금은 규정 하나를 직접 복사해서 넣었잖아요. 커넥트의 사내 문서가 인사규정, 보안규정, 복지규정, 출장규정... 수십 개입니다. 직원이 "보안 USB 규정이 뭐야?"라고 물으면 보안규정을 찾아서 넣어줘야 하고, "야근 식대 되나요?"라고 물으면 복지규정을 찾아서 넣어줘야 합니다.

질문할 때마다 사람이 관련 문서를 찾아서 복사-붙여넣기? 그건 AI 비서가 아니라 사람 비서입니다.

동료: "그걸 사람이 해주면 AI가 왜 필요해?"

맞는 말이었습니다. 이걸 자동으로 해줘야 합니다. 질문이 들어오면 수십 개의 문서 중에서 관련된 것만 자동으로 골라서 LLM에 넘겨주는 구조. 이 구조에 이름이 있습니다.

RAG(Retrieval-Augmented Generation). 직역하면 "검색으로 강화한 생성"인데요, 쉽게 말하면 이겁니다. 도서관에 사서를 한 명 앉혀놓는 겁니다. 누군가 질문하면 사서가 서가에서 관련 책을 찾아서 AI한테 건네주고, AI는 그 책을 읽고 답합니다.

지금부터 이 사서를 만들어보겠습니다.

RAG는 세 단계로 동작합니다.

첫째, 문서를 숫자로 바꿔서 저장합니다. 사람은 "연차"와 "휴가"가 비슷한 말이라는 걸 알지만, 컴퓨터는 모릅니다. 그래서 문서의 의미를 숫자 벡터로 변환해서 저장합니다. 이걸 임베딩(Embedding)이라고 해요.

둘째, 질문이 들어오면 질문도 같은 방식으로 숫자로 바꾸고, 저장해둔 문서들 중에서 가장 비슷한 것을 찾습니다. 숫자끼리 거리를 비교하면 의미가 비슷한 문서를 찾을 수 있습니다.

셋째, 찾은 문서를 LLM에 넘겨주고 답변을 생성합니다. step2에서 했던 것과 같은 원리인데, 문서를 찾는 과정이 자동화된 거예요.

사내 규정 3개를 문서로 만들어서 이 파이프라인을 직접 구성해보겠습니다.

# step3_rag.py — TODO: RAG 파이프라인 구성 from langchain_classic.chains import RetrievalQA from langchain_chroma import Chroma from langchain_ollama import OllamaEmbeddings, ChatOllama from langchain_core.documents import Document from langchain_core.prompts import PromptTemplate from rich.console import Console console = Console() # 1. 사내 규정 3개를 문서로 준비 docs = [ Document( page_content="[인사규정] 신입사원 휴가 및 연차: 신입사원은 입사 후 처음 3년 동안은 법정 연차가 발생하지 않습니다. 대신 매월 1회의 유급 '리프레시 데이'를 휴가로 사용할 수 있습니다.", metadata={"source": "인사규정"}), Document( page_content="[보안규정] 업무 보안: 모든 임직원은 회사에서 지급한 승인된 보안 USB만 사용해야 하며, 개인 USB나 외부 저장 매체 사용은 엄격히 금지됩니다.", metadata={"source": "보안규정"}), Document( page_content="[복지규정] 식대 지원: 점심 식사는 무제한 법인카드로 지원하며, 저녁 식사는 오후 9시 이후 야근 시에만 사용이 가능합니다.", metadata={"source": "복지규정"}), ]

인사규정, 보안규정, 복지규정. 세 개의 문서가 서가에 꽂혔습니다. 이제 이 문서들을 숫자로 바꿔서 검색 가능한 상태로 만들어야 합니다.

# 2. 문서를 벡터로 변환해서 ChromaDB에 저장 embeddings = OllamaEmbeddings(model="nomic-embed-text") vectorstore = Chroma.from_documents(documents=docs, embedding=embeddings) # 3. 질문과 가장 비슷한 문서를 찾는 검색기 생성 retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

nomic-embed-text는 문서의 의미를 숫자 벡터로 변환하는 임베딩 모델입니다. ChromaDB는 이 벡터들을 저장하고 검색해주는 벡터 데이터베이스예요. 사서에게 서가를 준 겁니다.

k=3은 "질문과 가장 비슷한 문서 3개를 가져와라"라는 뜻입니다. 지금은 문서가 3개뿐이니 전부 가져오겠지만, 나중에 문서가 수백 개가 되면 이 숫자가 중요해집니다.

마지막으로 검색기와 LLM을 하나의 체인으로 연결합니다. "질문 → 문서 검색 → LLM 답변"이 한 번에 돌아가는 파이프라인입니다.

# 4. 프롬프트 템플릿 template = """당신은 회사의 규정에 대해 설명해주는 AI 비서입니다. 아래의 참고 정보를 바탕으로 질문에 답하세요. 참고 정보: {context} 질문: {question} 답변:""" PROMPT = PromptTemplate(template=template, input_variables=["context", "question"]) # 5. 검색기 + LLM을 체인으로 연결 llm = ChatOllama(model="deepseek-r1:8b", temperature=0) qa_chain = RetrievalQA.from_chain_type( llm=llm, retriever=retriever, return_source_documents=True, chain_type_kwargs={"prompt": PROMPT} ) # 6. 질문 실행 question = "신입사원 휴가 규정에 대해 알려줘." result = qa_chain.invoke({"query": question})

엔터를 눌렀습니다.

실행 결과
--- 검색된 문서(근거) ---
[인사규정]: 신입사원 휴가 및 연차: 신입사원은 입사 후 처음 3년 동안은...
[복지규정]: 식대 지원: 점심 식사는 무제한 법인카드로...
[보안규정]: 업무 보안: 모든 임직원은...

--- AI 답변 ---
신입사원은 입사 후 처음 3년 동안은 법정 연차가 발생하지 않습니다. 대신 매월 1회의 유급 '리프레시 데이'를 휴가로 사용할 수 있습니다.

사서가 일을 했습니다. "신입사원 휴가"라는 질문을 받고, 서가에서 인사규정을 찾아서 AI에게 건네줬습니다. AI는 그 문서를 읽고 정확하게 답했습니다.

step1에서 근로기준법 60조를 지어내던 그 AI가, 지금은 커넥트의 실제 규정을 정확하게 전달하고 있습니다. 달라진 건 하나뿐입니다. 문서를 자동으로 찾아서 건네주는 사서, 즉 RAG 파이프라인이 생긴 겁니다.

동료: "오, 이거 진짜 되네. 근데 규정이 3개밖에 없잖아. 진짜 문서 수십 개 넣으면?"

좋은 질문입니다. 지금은 문서 3개를 메모리에 올려둔 것뿐입니다. 실제 사내 문서는 PDF, Word, Excel 등 다양한 형태로 수십~수백 개가 있습니다. 이 문서들을 어떻게 수집하고, 어떤 기준으로 자르고(청킹), 어떻게 벡터로 변환해서 저장할지 — 그 이야기는 다음 챕터부터 본격적으로 시작됩니다.

이것만은 기억하자

LLM은 모르는 것을 자신 있게 지어냅니다. 이걸 환각이라고 합니다. 문서를 직접 건네주면 해결되지만, 매번 사람이 찾아줄 수는 없습니다. RAG는 이 과정을 자동화한 구조입니다. 질문이 들어오면 관련 문서를 자동으로 찾아서 LLM에 넘겨줍니다. 도서관에 사서를 앉힌 겁니다.

다음 챕터에서는 이 사서가 답변할 데이터가 있는 사내 시스템을 먼저 살펴봅니다.
