그림 1-8 · LLM 단독 vs RAG

기존 PNG(01_llm-vs-rag.png)를 HTML로 이관. 노드 사이 공백은 flex space-between으로 양 끝까지 확장.

LR-V1 · 현행 유지형

2행 대시 박스 + 가로 플로우

PNG의 세로 스택 구조 유지, 색만 Primary/Secondary로 이관. 화살표가 flex-grow로 양 끝까지 뻗어 공백 제거.

LLM 단독
질문 → LLM → 환각 답변
RAG
질문 → 검색기 → 벡터 DB → LLM → 출처 포함 답변
▲ PNG와 동일 구조. 본문 수정 최소.
LR-V2 · Before/After

좌우 2열 카드 비교

단계 개수 자체(2 vs 3)가 시각적 대비. annotated-compare(그림 1-3)와 시각 언어 일관.

LLM 단독 호출 Hallucination
1. 질문을 받는다
2. LLM이 학습 데이터만으로 답변 생성
환각 답변 — 근거 없음
RAG 적용 Grounded
1. 질문을 받는다
2. 관련 문서를 검색해서 가져온다
3. LLM이 문서를 근거로 답변 생성
출처 포함 답변
▲ 카드 구조로 확장성 높음.
LR-V3 · 2갈래 분기

한 질문에서 두 경로로 갈라지기

경로 길이 차이로 "추가 단계"가 한눈에 보임. space-between으로 양쪽 끝까지 확장.

LLM 단독
질문 → LLM → 환각 답변
RAG
질문 → 검색기 → 벡터 DB → LLM → 출처 포함 답변
▲ 서사성 최강. 경로 길이 차이가 노골적으로 보인다.
LR-V4 · 매트릭스

단계 유무를 테이블로

빈칸(빗금)이 결핍을 직접 시각화. 튜닝 챕터 톤과 어울리고 공간 가장 적게 사용.

1. 질문 2. 검색 3. 생성 4. 결과
LLM 단독 질문 수신 — 없음 — LLM 생성 환각 답변
RAG 질문 수신 벡터 DB 조회 문서 + LLM 생성 출처 포함 답변
▲ 데이터성·공간 효율 모두 높음.
