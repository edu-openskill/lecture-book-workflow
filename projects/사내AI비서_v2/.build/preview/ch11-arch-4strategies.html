11.5 전체 구성도 — AI감 빼는 4가지 디자인 전략

현재 디자인(그라데이션·컬러 뱃지·섀도우)에서 AI스러운 신호를 걷어낸 4가지 방향을 제시합니다. 각 전략은 같은 구조(LLM + User·API / Runtime / Tools + Stores)를 유지하되 배경·선·타이포·색 언어만 바꿉니다. 네 개 모두 훑어보고 하나 선택해 주세요.

AI스러운 디자인의 전형
linear-gradient 과다 다색 뱃지(4색+) soft shadow 겹침 둥근 12~14px 코너 인디고 + 에메랄드 + 앰버 스팩트럼 아이콘 배지 + 원형 아이콘
반대 방향
단일 종이 배경 1~2색만 사용 hairline 보더 모노스페이스 + 이탤릭 여백으로 호흡 grid paper · 좌표

A. Editorial Paper — 출판물 · 잡지 완결 페이지

콘셉트: 책 마지막 장의 완결 요약 페이지. 아이보리 종이 위에 Serif Italic 헤더 + 모노 라벨. 장식은 hairline 보더와 코너 브래킷만. 색은 구리빛 산화동(oxide #a8553a) 단일 액센트로 .core와 사용자 아이콘만 포인트.
언제 어울리나: 책 전체 톤이 에디토리얼(출판물스러움)을 추구할 때. 잡지·단행본·학술서 마감 느낌.

B. Blueprint — 건축 도면

콘셉트: 기술 도면 스타일. 크림색 종이 + 전체 검정 외곽선 + 코너 브래킷. 모든 텍스트 모노스페이스 + 대문자. 색 완전 제거(검정만).
언제 어울리나: 기술서의 "전문 문서" 톤. 엔지니어링 다이어그램·P&ID 느낌이 책 정체성에 맞을 때.

C. Brutalist Print — 흑백 대비

콘셉트: 0색. 검정·흰색만. 3px 두꺼운 외곽선, 대문자 타이포, 타이포그래피가 주인공. .col-a와 .llm은 검정 반전 블록으로 강조. 섀도우·그라데이션·둥근 코너 전부 제거.
언제 어울리나: "강하고 단순한" 인상이 필요한 대담한 기술서. 독자 시선을 구조 자체에 집중시키고 싶을 때.

D. Minimal Single-Tone — 인디고 1색 · 여백 중심

콘셉트: 흰 배경 + 1px hairline 보더 + 브랜드 인디고 1색만. 섀도우·그라데이션·원형 아이콘 전부 제거. 넓은 여백(gap: var(--space-xl))이 호흡을 만든다. Runtime 컬럼만 인디고 보더로 강조.
언제 어울리나: 현 디자인 시스템(인디고 브랜드)을 유지하면서 AI감을 뺄 때. 가장 보수적이고 안전한 선택.

결정 가이드

전략 분위기 장점 단점 · 리스크
A출판물 · 잡지책 마지막 장답게 격조. 독자적 브랜드감Italic serif가 기술서에 생소할 수 있음
B도면 · 기술 문서엔지니어 정본 인상. 색 의존 0모노 전용이라 한글 가독성 약간 낮음
C브루탈리스트 · 흑백가장 강한 시각 인상. 결정적 마무리책 전체 톤과 이질감 있을 수 있음
D미니멀 · 인디고현 브랜드 유지 + AI감만 제거. 리스크 낮음가장 밋밋. "특별함" 약함

추천: 책의 에디토리얼 톤(캐릭터 대사·이야기 파트)과 궁합 좋은 A (Editorial Paper). 가장 안전하게 가려면 D (Minimal). 시각 충격이 필요하면 C (Brutalist). B (Blueprint)는 엔지니어링 문서 정체성을 강하게 가져갈 때.
