전체 구성도 — 챕터별 채워지는 시리즈

동일한 맵이 CH02부터 CH10까지 챕터를 지날 때마다 한 층씩 accent로 채워지는 progression. 독자가 각 챕터 끝에서 "내가 이만큼 왔구나"를 한눈에 체감합니다. CH08·CH09는 튜닝 챕터라 구성도가 없고, §10.10에서 완성본이 등장합니다.

범례: accent = 이번 챕터에서 추가 · 회색 = 이전 챕터에서 만든 것 (유지) · 흐림 = 앞으로 채울 것

진행감 요약

독자는 6개 장 끝에서 같은 지도를 다시 만나는데, 각 장마다 accent로 물든 영역이 한 층씩 넓어집니다. 책 전체에서 "내가 여기까지 왔다"가 시각적으로 확인되는 쌓임 시리즈.

  • CH02: 뼈대 2개 (FastAPI + Postgres)
  • CH04: + 파싱·벡터화 층
  • CH05: + 검색 층 (+ Ollama 연결)
  • CH06: + 에이전트 층 (PostgreSQL 연결 완성)
  • CH07: + 운영 래퍼 3종
  • CH10: + Vision 파서 + 평가 → 완성

CH08·CH09는 구성도 없음. 튜닝이 실제 구성 요소에 반영되는 건 CH10 최종본에 "튜닝 완료" 뱃지로 표시.

이 스타일을 적용하면

  • 각 챕터 §N.last(전체 구성도에서 챕터 N의 자리)가 이 스타일로 통일
  • Before/After 그림(5-4·6-4·7-2·10-2)과 같은 시각 언어(부모 박스 + 상단 뱃지 + accent/회색/점선 3단계)
  • 독자의 누적 성취감 + "다음 챕터가 어디를 채울지" 예고까지 포함
