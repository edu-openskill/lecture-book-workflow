엔진 흐름 다이어그램 — 3가지 스타일 비교

예시 엔진: CH05 LCEL 파이프라인 (Retriever → Prompt → LLM → Parser). "흐름이 실제로 흘러가는 게 보이도록" 세 가지 시각화 방식을 비교합니다. 선택 후 전 챕터(CH04·CH05·CH06·CH07·CH08·CH09·CH10 엔진)에 같은 언어로 적용할 예정입니다.

V2. 데이터 타입 흐름 (각 구간 중간값 표시) NEW

V3. 실제 데이터 흐름 (예시 따라가기) NEW · 가장 연상력 강함

각 단계에 실제 데이터 예시를 옆/아래에 달아 독자가 "질문 '휴가 규정 알려줘'가 들어가서 → 문서 3개가 검색돼 → 프롬프트에 끼워 넣고 → ..." 하는 실제 여정을 머릿속으로 따라감. 추상이 아니라 구체를 보여줘서 연상력이 가장 강함.

세 버전 비교

V1 — 현재 수평 파이프라인: 구조·이름만 보여주고 흐름감 약함. CH05에 이미 적용된 형태.

V2 — 데이터 타입 흐름: 각 블록 사이에 str · List[Document] · PromptValue · AIMessage · str 점선 뱃지. 화살표 위에 진행감 있는 큰 라인. 프로그래머 친화적으로 타입 변화를 보여줌. 차지 면적 작음.

V3 — 실제 데이터 예시 흐름: 각 단계에 실제 JSON·문자열을 달아 놓음. "질문 '휴가 규정 알려줘'가 들어가서 → 문서 3개가 나와 → 프롬프트에 들어가 → LLM 답변 객체 → 최종 문자열" 여정을 눈으로 따라감. 연상력 가장 강함. 세로로 길지만 기술 파트 설명에 적합.

선택 가이드

  • 구조 개요만 (이야기 파트 / prep) → V1 또는 V2
  • 엔진 동작을 설명할 때 (기술 파트) → V3 강력 추천. 실제 데이터가 머릿속에 남음
  • 공간 제약 있을 때 → V2 (가로 배치, 한 장에 들어감)
  • 혼합 (권장): 챕터 도입부에 V2로 구조·타입을 먼저 보여주고, 코드 섹션에서 V3로 실제 여정을 따라가는 2단 구성. CH05처럼 "개념 → 구현" 순서 있는 챕터에 자연스러움.
