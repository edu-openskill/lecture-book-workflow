11.5 A4 세로 레이아웃 4종 — d1-tokens 에디토리얼

A4 본문 폭(~658px) 안에 맞추는 4가지 세로 배치. LLM과 MCP Tools를 위쪽에 배치하고 Runtime·User·Stores를 아래로 흐르게. 디자인은 d1-tokens의 에디토리얼(아이보리 종이, Serif italic, 브라켓 박스) 스타일을 프로젝트 토큰으로 번역.

V1. 상단 2열: LLM ┃ Tools

상단 2칸 분할 LLM 좌 · Tools 우 수직 4단 흐름

구조: [LLM | Tools] 2칸 나란히 → Runtime → User·Gateway → Stores. 상단에서 "외부 자원" 두 개를 동시에 제시, 아래로 실행 경로.

V2. LLM 가로 전체 + Tools 가로 전체 (2행 분리)

LLM 1행 전체 Tools 1행 전체 (4열) 서로 구분되는 2행

구조: LLM 한 줄, Tools 한 줄. 각자 가로 폭 전체를 씀. 외부 자원 2종을 나란히 나열하되 서로 독립된 행으로 분리. 시선이 LLM → Tools → Runtime으로 자연스럽게 내려옴.

V3. External Services 그룹 (LLM + Tools 묶음)

V4. 세로 단일 컬럼 5행 스택

LLM / Tools(2×2) / Runtime / User·API / Stores 모든 레이어 가로 전체 계층 흐름 가장 명확

구조: 위에서부터 LLM → Tools(2×2 그리드) → Runtime → User·Gateway → Stores. 각 레이어가 가로 폭 100% 씀. Tools가 4개라 2×2로 컴팩트 배치.

V4 · 세로 단일 컬럼 5행. LLM·Tools·Runtime·User·Stores가 계층적으로 내려오는 가장 단순 구조

결정 가이드

레이아웃 상단 배치 강점 단점
V1 2열 상단 LLM ┃ Tools 가로 2칸 균형, 공간 효율 Tools 내용이 좁음
V2 2행 전체 LLM 한 줄 + Tools 한 줄 각 레이어 내용 여유 세로 길이 증가
V3 External 그룹 ★ External Services 외곽 박스 + LLM·Tools 2칸 "외부 의존" 개념 강조. 에이전트와 외부 분리 명확 중첩 박스라 구조 약간 복잡
V4 세로 5행 LLM·Tools·Runtime 각자 한 줄 흐름 가장 명확, 계층 분명 세로 길이 최장. LLM·Tools가 대등한 레이어처럼 보임

추천: V3 External Services 그룹. "LLM과 Tools가 둘 다 에이전트 외부의 리소스"라는 개념이 외곽 박스 하나로 명시되어 독자 이해도가 높음. V2는 안전한 대안.
