Runtime 컬럼 톤 조정 프리뷰 — 4단계

D 디자인의 Runtime(에이전트 실행 레이어) 컬럼만 4단계 톤으로 조정. LLM·User·Tools·Stores는 고정. Runtime 컬럼의 채도·명도·대비만 T1 → T4로 점점 올라갑니다.

T1. Whisper — 가장 연함 (현재 상태)

대비 0.5 / 채도 0 / 명도 최고. Runtime 컬럼이 다른 컬럼들과 거의 구분이 안 됨. 타이틀은 muted 회색. Core·Engine 모두 hairline만. 인디고는 engine 좌측 2px bar 1곳에만.

border 1px #e2e8f0 bg white title: muted gray core: hairline engine: 2px accent bar only

T2. Soft Hint — 포인트 힌트 (연중)

대비 1 / 채도 1 / 명도 높음. Border는 `--accent-border`(#c7d2fe) 연한 인디고. 타이틀·라벨에 accent 색 적용. Core는 accent 외곽선. Engine은 좌측 3px + 외곽 연한 accent-border.

border 1px #c7d2fe (accent-border) bg white title: accent 인디고 텍스트 core: accent solid border engine: accent-border + 3px bar

T3. Balanced — 균형 (중간, 추천)

대비 2 / 채도 1.5 / 명도 중간. 상단 4px accent bar로 중심 명시. 타이틀은 accent border 노치. Core·Engine 배경에 미세한 accent tint(#f8f9ff)로 따뜻한 인디고 워머. stage 칩은 accent-bg 연파랑 채움. User/Tools 블루와 밸런스 맞음.

border 1.5px + top 4px accent bar bg white title: accent hairline chip core: accent-bg + accent border engine: #f8f9ff micro tint

T4. Featured — 전경 (진함, 센터 강조)

border 2px accent bg #f8f9ff (≈2% accent tint) title: accent solid chip core: accent solid, white text engine: accent-bg stages + 4px bar

결정 가이드

톤 채도·명도·대비 핵심 시각 신호 어울리는 상황
T1 Whisper 채도 0 · 명도 最高 · 대비 0.5 회색 hairline, engine 좌측 2px bar만 인디고 User/Tools만 존재감 있으면 됨. Runtime은 "배경"
T2 Soft Hint 채도 1 · 명도 高 · 대비 1 accent-border 연한 인디고 보더 + 인디고 텍스트 톤 맞추면서 Runtime 정체성 약간만 살리고 싶을 때
T3 Balanced ★ 채도 1.5 · 명도 中 · 대비 2 상단 4px bar + accent 텍스트 + Core 연한 tint 세 컬럼 밸런스 최적. Runtime 중심성 유지
T4 Featured 채도 2 · 명도 低 · 대비 3 컬럼 전체 tint + solid accent Core Runtime이 "엔진"임을 강하게 외치고 싶을 때

추천: T3 Balanced (★ 행). 상단 4px bar + 흰 배경 + 미세 tint Core가 User·Tools 블루와 조화를 이루면서 Runtime이 중앙 엔진임을 분명히 합니다. T2도 좋은 대안.
