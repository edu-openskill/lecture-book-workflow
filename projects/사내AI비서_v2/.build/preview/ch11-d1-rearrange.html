11.5 d1-tokens 스타일 · LLM+Tools 상단 배치 4안

fig-arch-d1-tokens.html의 CSS와 타이포그래피를 그대로 유지. LLM과 MCP Tools만 위로 올리고 Client(User+API) · Agent · Stores는 그대로 둠. 모두 A4 본문 폭(~658px)에 맞춘 축소 렌더링(.d1.a4) 적용. Runtime(Agent) 컬럼에는 이전 요청대로 ReAct Loop와 커넥트HR 엔진이 Executor 안에 중첩.

결정 가이드

안 상단 배치 강점 단점
V1 LLM ┃ Tools 가로 2칸 균형. 공간 효율 최고 각 박스 내용 약간 좁음
V2 LLM 전체 행 + Tools 전체 행 각 레이어 내용 여유 세로 길이 증가
V3 ★ External Services 외곽 박스 + LLM·Tools 2칸 "외부 의존" 개념 그루핑. 에디토리얼 콜아웃 느낌 중첩 박스라 구조 복잡
V4 Tools 좌 ┃ LLM 우 Agent→Tools 시선 연결 LLM이 주 역할인데 오른쪽 놓임

추천: V3 External Services 외곽 그룹. d1-tokens 스타일의 상징인 ::before/::after 브라켓 장식과 어울리는 "외부 서비스 콜아웃" 그루핑. V1이 가장 단순한 대안.
