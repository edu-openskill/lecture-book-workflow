시스템 아키텍처 — 리서치 기반 배치 3종

C4 Container Model(Simon Brown) · AWS/Azure 참조 아키텍처 · Grokking System Design · Data Flow Diagram(Yourdon) 원칙을 종합해 독자가 빠르게 이해할 수 있는 3가지 배치를 제안합니다. 챕터 분류 대신 시스템 책임과 통신에 초점.

리서치 기반 공통 원칙
  1. 경계 박스로 책임 구분 (C4 Container) — Online serving / Offline ingestion / External / Data store
  2. 화살표에 protocol·data 라벨 — "HTTP POST JSON", "Vector Search k=5", "SQL Query", "Prompt + Context"
  3. 메인 요청 경로 강조 — 굵은 파란 선, 보조 흐름은 얇은 회색·점선
  4. 좌→우 · 위→아래 시선 이동 — 한국어 독자에게 자연스러운 동선
  5. 모양으로 요소 타입 구분 (DFD) — 외부 엔티티 = 사각형, 프로세스 = 둥근 사각형, 데이터 저장소 = 평행선
  6. 범례 필수 — 색·선 스타일 의미 하단에
  7. 노이즈 최소 — 비핵심 요소는 생략, 필요한 것만

R1. C4 Container 스타일 (경계 + 라벨 화살표) 가장 표준적

C4 Level 2 Container Diagram 원칙. 4개 경계 박스(Online · Data · Ingestion · External)로 책임을 구분하고, 각 경계 안에 컨테이너(FastAPI, Agent 등)를 배치. 화살표 라벨로 통신을 명시. 레퍼런스: c4model.com, Simon Brown "Software Architecture for Developers".
