화살표 전략 3종 프리뷰

에이전트·LLM·도구·FastAPI 사이 관계를 화살표로 전달하는 3가지 방식. 같은 구성(CH10 완성본)에 화살표만 다르게 적용해 비교합니다.

전략 A · 번호 시계열 ①~⑥

모든 화살표에 순서 번호 뱃지. 독자가 ①→②→③ 따라가며 여정을 읽음. 하단에 순서 범례.

전략장점단점
A · 번호시간순 명확 · 독자가 숫자로 읽기라벨 많아지면 시선 이동
B · 색상·스타일유형 즉시 인식 · 깔끔범례 기억 필요
C · 데이터 라벨통신 내용 투명 · 프로그래머 친화칩 많아지면 시각 혼잡
A + C 하이브리드순서 + 데이터 동시 · 여정 이해 최강요소 밀도 약간 상승
