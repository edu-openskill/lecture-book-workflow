### 4.2 충돌·실종 정리

#### C-실종 1: Docker 공장 → K8s 항만 관제 센터 비유의 폐기

CH01 L83 표가 K8s를 "수많은 배를 관리하는 항만 관제 센터"로 못 박는데, CH04 L37에서 **"이번 챕터부터는 더 가까운 비유인 프랜차이즈 본사로 풀어 봅니다"** 라고 메타 서술로 갈아치운다. 비유 두 개가 K8s에 동시 적용 → **충돌**.

#### C-실종 2: outline-v2가 약속한 "톨게이트 vs 안내데스크" 비유 실종

outline-v2 L233~234:
> "5.2.2 L4 vs L7 ← 파일02 기반
> - **톨게이트(L4: IP/Port만) vs 안내데스크(L7: URL/Host까지)**"

**현실** CH05 v11에는 5.2.2가 없고 5.2.1, 5.2.2, 5.2.3 구조이며, L4/L7 비유는 L481 참고 박스에 "봉투 겉면 정보를 다루는 계층을 L4(전송 계층), 봉투 안쪽 정보를 다루는 계층을 L7(애플리케이션 계층)"이라는 **봉투 비유로 대체**됐고 **톨게이트·안내데스크 단어는 0회**. chapters-v3 검수 L181이 "고속도로 분기점 vs 안내데스크" 비유를 책 전체 백미로 칭찬했던 부분이 **사라짐**.

**영향**: chapters-v3 검수가 A 등급 결정타로 꼽았던 비유가 v11에서 빠지면서 5.3 전체 흐름의 클라이맥스가 약해짐.

#### C-실종 3: Docker DNS → CoreDNS 매핑의 명시적 단어 부재

outline-v2 L254 표가 약속한 "Docker DNS (127.0.0.11) → CoreDNS"의 매핑이 CH05/CH06 본문에서 **CoreDNS 단어 자체가 0회**. CH05 L411 "전용 DNS 서버", L420 "쿠버네티스는 클러스터 안에 DNS 서버를 함께 띄워 둡니다" 라고만 표현. CH06 L326 "클러스터 DNS"만 언급.

CH02 L304~305 "**Docker DNS**" 단어는 분명히 명명. 그런데 "이게 K8s에서는 CoreDNS다"라는 명시적 매핑이 없어 독자가 둘을 같은 부류로 인식하기 어렵다.

**수정 방향**: CH05 L420 참고 박스에 "Docker에서는 Docker DNS, K8s에서는 **CoreDNS**라는 이름의 DNS 서버가 같은 일을 합니다" 한 줄 추가.

#### C-실종 4: outline-v2 L249~258 "Docker → Kubernetes 매핑 표"가 v11에 없음

outline-v2가 약속한 6행 매핑 표(docker0/iptables DNAT/Docker DNS/NGINX 라우팅/docker network create/Compose 자동) **CH05 v11에 없음**. v11 L490~494 "이것만은 기억하자"는 Service/Ingress 요약만. chapters-v3 검수 L182~183가 **"5.3.2 Docker→K8s 매핑 표 — 책 전체 백미"**로 칭찬한 표가 사라짐.

**영향**: outline-v2의 책 통합 클라이맥스가 빠진 것. 독자가 "Docker에서 배운 게 K8s에서 어떻게 변했지?" 를 한 표로 보지 못함.

**수정 방향**: CH05 5.3.2를 신설(이미 outline-v2에 명시됨)하거나 5.3 끝에 6행 표 삽입.

### 4.3 비유 동시 적용 혼선

CH04 L184~191의 "Pod = 가맹점" 비유와 CH04 L60~68의 표(가맹점=Pod, 본사=컨트롤 플레인, 상가 건물=워커 노드)는 일관. 그러나:

- **CH04 L82 "워커 노드: 가맹점이 위치한 상가 건물"** — 워커 노드 = 상가 건물
- **CH04 L184 "Pod는 가맹점 하나와 같습니다"** — Pod = 가맹점

여기까진 OK. 하지만 **CH05 L25 "가맹점 안에서 직원이 교대 근무를 해도 고객은 직통 전화번호 하나로"** — 여기서 "직원"이 **컨테이너**인지 **Pod**인지 모호. CH04에서는 "**가맹점 안의 직원(컨테이너)**" (L184)이라고 컨테이너=직원으로 매핑했는데 CH05에서 직원이 **교대 근무**한다는 표현이 IP가 바뀌는 Pod의 행동에 대응. **컨테이너가 교대 근무하는 게 아니라 Pod 자체가 새 IP로 교체되는 것**이라 비유 정확도 약함.

**수정 방향**: CH05 L25 "**가맹점이 자리를 옮겨 다녀도(IP가 바뀌어도) 고객은 본사 직통 전화번호 하나로**" 정도로 정정.
