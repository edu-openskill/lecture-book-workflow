## 1. 목적

현재 원고는 **AI가 오픈이를 관찰하며 학습 진도를 요약하는 3인칭 해설 모드**로 작동한다. 오픈이가 실제로 실습하는 1인칭 현장감이 부족하고, 메타 서술자 어미("그림이 그려졌습니다", "감이 잡혔습니다", "머리에 정리됩니다")가 각 절 말미를 도배하고 있다.

이 문서는 6개 챕터 전수 분석을 통해:
1. 공통된 AI 서술 패턴을 목록화하고
2. 챕터별 구조적 약점을 표로 대조하고
3. 재작성 우선순위와 처방 원칙을 세운다.

**라벨 정의**

- **FILLER**: 불필요한 부연/수식 (예: "입문 자료에 공통으로 등장하는 nginx")

## 6. 처방 원칙

### 6.1 서술자 목소리 재설정
- **현재**: AI가 오픈이를 위에서 관찰하며 "이제 이해했다"고 해설하는 3인칭 요약자
- **목표**: 오픈이 어깨 너머 카메라. 오픈이가 **치고 · 보고 · 느낀 것**만 서술
- **내면 독백**은 오픈이 본인의 1인칭 목소리 — 메타 해설과 구분

### 6.2 금지 표현 표 (일괄 정리 대상)

| BAD (제거/치환) | GOOD (대안) |
|----|----|
| "그림이 그려졌습니다 / 잡혔습니다" | 삭제하거나 → 오픈이가 노트에 적은 장면 |
| "감이 잡혔습니다 / 왔습니다" | 삭제하거나 → `*'이제 ~는 안 까먹겠네.'*` 독백 |
| "머리에 정리됩니다 / 손에 들어왔습니다" | 삭제하거나 → 의자 뒤로 젖힘 / 커피 한 모금 |
| "이 절의 핵심이었습니다" | 삭제 (다음 장면으로 바로) |
| "~차례였습니다 / 자리였습니다 / 단계였습니다" | 삭제 (다음 명령어부터 바로) |
| "결론은 한 줄로 수렴됐습니다" | 삭제 |
| "~장면이 말이 됐습니다 / 장면이 나왔습니다" | 삭제 또는 구체 행동 |
| "~할 필요가 있었습니다" | 삭제 또는 "~해야 했다" 직접형 |
| "~편이 나았습니다 / 좋았습니다" | 삭제 또는 "~했다" 직접형 |
| "~해 보였습니다 / 적당해 보였습니다" | 삭제 또는 직접 판단 |
| "~셈이었습니다 / 얘기였습니다" | 해당 사실을 직접 진술 |

### 6.3 구조적 보강 (챕터 기본 요건)

각 챕터 최소치:
1. **내면 독백** `*'...'*` — 섹션당 1회 이상
2. **감각 묘사** — 섹션당 1개 (시각 + 청각/촉각 조합)
3. **Try/Fail** — 챕터당 1회 이상 (**예제 코드 기반**으로만. 지어내지 말 것)
4. **캐릭터 대사** — 챕터당 1회 이상, 2챕터 연속 부재 금지

4. 각 챕터는 **예제 코드 기반 실행 내용만** 유지. 실패/장면 지어내기 금지 규칙 재확인

## 8. 부록: 라벨 사용 가이드

### META — 서술자 요약 목소리
문장 끝이 "~셈이었습니다 / ~모양이었습니다 / ~얘기였습니다 / ~였습니다(일반 사실 아닌 요약)" 로 끝나고, 오픈이 학습 상태를 AI가 외부에서 정리하는 어투. **가장 많이 잡아야 할 패턴.**

### HEDGE — 약한 평가/판단
확신을 회피하는 어미: "~편이 나았습니다 / ~해 보였습니다 / ~할 것 같았습니다 / ~할 필요가 있었습니다 / ~ㄹ 터였습니다". 필자가 "적절해 보이게" 쓰고 싶을 때 나오는 AI 버릇.

### SCAFFOLD — 학습 진행 해설/전환
"이제 ~할 차례였습니다 / ~단계였습니다 / 여기서 정리됐습니다 / 오늘 과제 중 ~가 풀렸습니다". 섹션 말미마다 AI가 "독자님, 지금 여기까지 왔습니다" 알려주는 목소리.

### FILLER — 불필요한 부연/수식
의미 변화 없이 문장만 늘리는 구절. "입문 자료에 공통으로 등장하는", "되짚어 보았습니다" 같은 관행적 수식.

### WEAK-STORY — 구조적 약점
개별 문장이 아니라 **섹션/챕터 전체 구조**의 결손. 캐릭터·독백·감각·실패·원형서사 등.
