5.3.2 본사 + 가맹점 큰 그림 시안 v2 (콜센터 매핑)

v1 → v2 변경 사항
Ingress Controller 비유: 본사 안내데스크 → 본사 콜센터
이유: CH04에서 "본사 안내데스크 = Kube API Server"가 이미 자리잡고 있어 충돌. "Service = 가맹점 직통 전화번호"(5.1.1)와의 통화 결을 잇기 위해 콜센터로 변경.
호텔 단어 폐기. 본사·가맹점 비유 안에서 풀이. 본사 빌딩 단면도(가운데) + 외부 가맹점들(오른쪽)이 한 도화지에. 본사 안에 정문·안내직원·콜센터·가맹점 명부·본사 인사팀이 모두 자리. 외부 손님은 본사를 거쳐 가맹점에 닿음.

큰 그림 시안 v2

그림 안 v2. 본사 빌딩 안 정문·안내직원·콜센터·가맹점 명부, 외부에 위치한 가맹점들. 본사 인사팀은 가맹점 변동을 감시해 명부 갱신
v2 점검 포인트
  • "본사 콜센터"가 헤드셋·전화기·통화 부스로 시각화되어 안내데스크와 구별되는지
  • 5.1.1의 "Service = 가맹점 직통 전화번호"와 "콜센터 → 가맹점 전화 연결" 흐름이 자연스럽게 이어지는지
  • CH04의 "Kube API Server = 본사 안내데스크"와 충돌이 없는지 (콜센터는 다른 단어·다른 시각요소)
  • 콜센터 ↔ 가맹점 명부 점선 양방향(이름 질의 → 위치 응답)이 자연스러운지
  • 콜센터 → 외부 가맹점 3개 분기 화살표 라벨이 "전화 연결"로 통화 결을 살리는지
  • 본사 빌딩(가운데) + 외부 가맹점들(오른쪽) 배치는 v1 그대로 유지

이 시안 OK면 본문 작업으로 넘어갑니다 — 5.2.2의 "공식 앱" → "본사 콜센터" 본문 다듬기, 5.3.2 STEP 1~5 비유 단락 본사·콜센터·가맹점 안에서 정리.
