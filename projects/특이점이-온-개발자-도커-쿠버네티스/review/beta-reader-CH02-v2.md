## 심각도별 이슈

### 높음 (독자가 막힘)

1. **캐릭터 대화가 챕터 전체를 관통 — 저자 규칙 충돌**
   - 오픈이와 선배가 처음부터 끝까지 대화하며 개념을 전달합니다. 저자 피드백에 따르면 "캐릭터는 프롤로그에만, 챕터 이야기 파트는 비유+그림+예시로 개념 설명"이 규칙입니다. 현재 구조는 이 규칙과 정면으로 충돌합니다.
   - **저자 결정 필요**: (A) 캐릭터 제거 후 비유+그림+예시 구조로 전환, 또는 (B) 규칙 변경하여 챕터에서도 대화체 허용.

2. **2.2 전체 그림 섹션 — 실습 전 고급 개념 과부하**
   - Network Namespace, veth pair, docker0 브릿지, iptables DNAT, Docker DNS 등이 실습(2.3~2.4) 전에 한꺼번에 등장합니다. Docker를 한 번도 써보지 않은 독자에게 네트워크 내부 구조를 먼저 보여주는 것은 순서가 맞지 않습니다.
   - 입문자 페르소나 4명(김지윤, 오새봄, 강민서, 한재원)이 이 섹션에서 포기 지점을 경험합니다.
   - 제안: 2.4 실습 이후로 이동 / 별도 심화 섹션으로 분리 / 그림 1장 + 요약으로 축소.

### 중간 (혼란스럽지만 진행 가능)

3. **`---` 수평선 7곳 잔존**
   - v1에서 지적된 이슈입니다. style.md 금지 규칙 위반이며 아직 미개선입니다.

4. **2.5 Linux 섹션 — 15종 명령어 과부하**
   - pwd, cd, ls, mkdir, touch, rm, cp, mv, apt, vim, ps, kill, find, tail, netstat 등 15종 명령어가 한 번에 다뤄집니다. Docker 이해가 목적인 챕터에서 Linux 입문서 수준의 분량이 삽입되어 있습니다.
   - "이미 리눅스에 익숙하다면 건너뛰어도 됩니다" 안내가 없어 경험자는 지루함, 입문자는 압도감을 느낍니다.

5. **컨테이너 ID 안내 부족**
   - `d2b1`, `5fcd` 등 축약 ID가 예시로 사용되지만, "독자의 컨테이너 ID는 다릅니다. `docker ps`로 확인한 ID 앞 4자리를 사용하세요"라는 안내가 없습니다. 처음 접하는 독자가 예시 ID를 그대로 입력하면 오류를 만납니다.

### 낮음 (개선하면 좋음)

6. **바인드 마운트 OS별 명령어 분리 시 안내 부족**
   - Windows/Mac/Linux별로 마운트 명령어가 다른데, 독자가 자신의 OS에 맞는 명령어를 선택하는 안내가 명확하지 않습니다.

## 페르소나별 상세

### 1. 이준혁 (백엔드 3년, Java/Spring)

**[이야기]** 오픈이/선배 대화 형식이 서사적으로는 읽기 편하지만, 캐릭터 대화가 기술 설명의 깊이를 약화시키는 부분이 있습니다. 비유(붕어빵 틀 = 이미지)는 명확합니다.

**[기술]** 커널 공유 개념, 이미지/컨테이너 구분, exec vs attach 비교가 논리적입니다. 2.2 네트워크 전체 그림은 이 시점에서 이해 가능하지만 실습 전에 나오는 것이 아쉽습니다.

**[실습]** 명령어 흐름이 논리적. v1에서 지적된 dockerhub ID 하드코딩은 개선되었습니다.

**[총평]** 기술 내용 자체는 탄탄합니다. 캐릭터 규칙 충돌이 편집 관점에서 가장 큰 이슈입니다.

**[점수]** 이야기 3/5 | 기술 4/5 | 실습 4/5

**[실습]** 전반적으로 무리 없이 진행 가능합니다. 포그라운드 관련 사전 안내가 v1 대비 개선된 점이 좋습니다.

**[총평]** 무난하게 따라갈 수 있습니다.

### 3. 김지윤 (프론트엔드 3년, React)

**[이야기]** 오픈이/선배 대화는 가볍게 읽히지만, 2.2에서 갑자기 네트워크 심화 개념이 쏟아지며 몰입이 깨집니다.

**[기술]** 커널 설명까지는 따라갔으나, veth pair, docker0, iptables DNAT은 프론트엔드 배경으로는 이해 불가입니다. 이 부분이 필수인지 선택인지 표시가 없어 건너뛸 수 없습니다.

**[실습]** 기본 docker run/stop/rm은 따라갈 수 있습니다. Linux 명령어 섹션은 부담스럽지만 컨테이너 내부 실습에 필요하다는 건 이해합니다.

**[총평]** 2.2 네트워크 섹션이 병목입니다. 이 섹션을 선택적으로 읽을 수 있게 하면 조건부 통과 → 통과로 올라갈 수 있습니다.

**[점수]** 이야기 3/5 | 기술 2/5 | 실습 3/5

**[실습]** Linux 명령어 15종을 한 번에 배우는 것이 Docker 학습 목적과 거리감이 있습니다.

**[총평]** 2.2와 2.5 두 곳의 난이도 장벽이 있습니다.
