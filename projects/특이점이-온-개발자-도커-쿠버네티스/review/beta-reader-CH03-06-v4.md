## 심각도별 이슈

### 높음 (학생이 막힘)

1. **CH03 3.4 이후 이야기 단절**
   - 위치: CH03 MySQL~Docker Compose 구간
   - 증상: 오픈이/팀장/동료 캐릭터가 완전히 사라지고 튜토리얼 모드로 전환
   - 영향: PM/비전공자는 이야기 흐름을 잃고 기술 파트와 구분 불가
   - 제안: MySQL 도입 동기를 오픈이의 문제 상황(채팅 기록 저장)으로 연결, 동료/팀장 대사 1~2개 추가

2. **CH05 삼중 포트 구분**
   - 위치: CH05 Service 유형 비교 구간
   - 증상: ClusterIP(클러스터 내부) → NodePort(노드 포트 개방) → LoadBalancer(외부 IP)의 계층적 관계가 한 번에 쏟아짐
   - 영향: 비개발 배경 독자 3명이 "셋 다 비슷해 보인다"고 응답
   - 제안: 점진적 빌드업 — "ClusterIP만으로는 밖에서 못 들어온다 → 그래서 NodePort → 그래도 IP 외우기 싫다 → 그래서 LoadBalancer" 흐름 강화

3. **CH06 DB Deployment without StatefulSet 경고**
   - 위치: CH06 종합실습 MySQL YAML
   - 증상: DB를 Deployment로 배포하는데 StatefulSet과의 차이/한계 언급 없음
   - 영향: CTO/DevOps가 "실무에서 이대로 쓰면 데이터 유실" 우려
   - 제안: 참고 박스로 "실무에서는 StatefulSet을 쓴다. 여기서는 학습 목적으로 Deployment를 사용한다" 한 줄 추가

### 중간 (혼란스럽지만 진행 가능)

4. **CH05 Ingress 실습 부재**
   - Ingress 개념은 설명하지만 실습은 CH06으로 이관됨
   - 독자 5명이 "읽기만 하고 넘어가니 불안하다"
   - 제안: CH05 끝에 "다음 챕터 종합실습에서 직접 Ingress를 구성한다" 예고 강화

5. **CH05 kube-proxy/iptables 깊이**
   - PM/비전공자에게 DNAT, iptables 규칙은 과도
   - 제안: 핵심 한 줄("트래픽을 Pod에 전달하는 내부 장치") + 참고 박스로 분리

6. **CH06 이야기 파트 빈약**
   - 종합실습 진입 후 캐릭터/비유 없이 YAML 나열
   - 제안: 종합실습 시작 시 오픈이가 "지금까지 배운 걸 모아서 한 번에 띄워보자" 식 동기 부여 1단락

7. **CH03 Dockerfile 비밀번호 하드코딩**
   - `MYSQL_ROOT_PASSWORD=1234` 가 그대로 노출
   - 제안: 코드 아래 참고 박스 — "실무에서는 환경변수나 Secret을 사용한다. 여기서는 학습용으로 직접 넣었다"

8. **CH06 Ingress 기반 접속 미시연**
   - 종합실습에서 NodePort로만 접속, Ingress 경로 라우팅 미시연
   - 제안: 선택 실습으로 Ingress 추가 구성 제시 (필수 아닌 보너스)

### 낮음 (개선하면 좋음)

   - "pod에 대한 하는" → 문법 오류
   - 제안: "Pod에 대한 설정" 또는 맥락에 맞게 수정

10. **CH05 L261 "스스로" 오타**
    - "스스로" → "스스로" (ㅡ 탈락) 확인 필요
    - 제안: "스스로" 맞춤법 확인

    - Dockerfile에서 CMD가 ENTRYPOINT보다 먼저 등장
    - 기능상 문제 없으나 관례와 다름
    - 제안: 교육적 관점에서 ENTRYPOINT → CMD 순서로 재배치하거나 순서 무관 언급
