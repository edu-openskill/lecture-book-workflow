## 심각도별 이슈

### 높음 (독자가 막힘)

1. **이야기 파트 부재 / 재성 캐릭터 미등장**
   - 챕터 전체가 기술 설명 + 실습으로만 구성되어 있음. 1.1(말콤 맥린) 섹션이 이야기 파트에 해당하나, 등장인물인 재성이 전혀 나오지 않음. 책의 핵심 컨셉("이야기처럼 쓰는 책")과 어긋남. `style.md`의 "이야기 먼저" 원칙 불이행.
   - 재성이 실제 환경 오류를 겪으며 Docker를 도입하는 흐름이 없으면, 독자가 "왜 이걸 배워야 하는가"를 감정적으로 납득하기 어려움.

2. **docker commit 실습 코드에 저자 Dockerhub ID 하드코딩**
   - `docker commit 5fcd coderyu5523/tomcat` — 독자가 그대로 복사-붙여넣기 하면 본인 계정이 아닌 저자 계정에 push를 시도하게 됨. `<본인-dockerhub-id>/tomcat` 형태로 변경 필요.

### 중간 (혼란스럽지만 진행 가능)

3. **`---` 수평선 반복 사용**
   - 1.2.4 이후, 1.4 명령어 표 이후, 1.8 마운트 마무리 직전 등 여러 곳에 `---` 수평선이 등장. 특히 "이것만은 기억하자" 바로 위에 `---`가 연속 2번(`---\n\n---`) 나타남. `structure.md` 및 `style.md` 금지 규칙 위반.

4. **1.5 Linux 섹션의 위치 및 분량**
   - Docker 개념 챕터 안에 Linux 명령어 레퍼런스가 독립 챕터 수준으로 삽입되어 있음. pwd, cd, ls, mkdir, touch, rm, cp, mv, apt, vim, ps, kill, find, tail — 14종 명령어를 한 번에 다루는 것은 분량 과부하. 이미 아는 독자(백엔드, 시니어)는 지루함을 느끼고, 처음 접하는 독자(부트캠프, PM)는 압도감을 느낌.
   - "이미 리눅스에 익숙하다면 1.6으로 건너뛰어도 됩니다" 안내가 있으나, 건너뛰면 실습 흐름이 끊기는 구조(1.5에서 Ubuntu 컨테이너를 실행하고 nginx를 띄운 뒤 1.6에서 해당 컨테이너를 계속 사용).

### 낮음 (개선하면 좋음)

5. **컨테이너 ID 부분 일치 표기 혼용**
   - `5fcd`, `d2b1` 등 축약 ID를 예시로 사용하는데, 독자에게는 자신의 컨테이너 ID가 다를 수 있다는 안내가 없음. 처음 독자는 예시 ID를 그대로 입력하다가 오류를 만날 수 있음.

6. **포그라운드/백그라운드 개념 도입 시점**
   - 1.4.3에서 `docker run nginx`를 실행하면 터미널이 포그라운드로 잠기는 현상이 발생하나, 이에 대한 사전 안내 없이 실행 후 설명됨. 처음 접하는 독자는 터미널이 멈춘 줄 알고 당황할 수 있음.

7. **`docker login` 실습에서 `ENTER 입력` 텍스트**
   - 코드 블록 안에 `ENTER 입력`이라는 한글 설명이 코드와 혼재. 코드 블록은 실행 가능한 명령어만 담아야 하며, 절차 설명은 코드 블록 밖으로 분리 필요.

8. **이야기 파트와 기술 파트 전환 문장 미비**
   - 1.1(말콤 맥린)에서 1.2(Docker 작동 원리)로 넘어갈 때 자연스러운 전환 문장 없이 `## 1.2 Docker : 작동 원리` 헤더가 바로 등장. 말콤 맥린 이야기와 기술 설명 사이의 연결고리가 약함.

### 1. 이준혁 (백엔드 개발자 3년차, Docker 처음)

**[이야기]** 말콤 맥린 이야기는 배경지식 없이도 읽히고, "내 컴퓨터에서 됐는데 서버에서 안 돼"라는 문제의식이 자신의 실무 경험과 바로 연결되어 공감됨. 다만 재성이라는 주인공이 한 번도 안 나와서 책인지 문서인지 모르겠음.

**[기술]** 커널 개념 설명이 군더더기 없이 명확. 이미지 = 붕어빵 틀 비유는 단박에 이해됨. `docker commit`에서 `coderyu5523` 하드코딩 발견 — 실무자라서 바로 알아챘지만 처음 접하는 독자는 헷갈릴 것.

**[총평]** 기술 내용 자체는 탄탄하나, 이야기 파트 부재로 책이 아닌 튜토리얼 문서처럼 느껴짐.

### 8. 임성현 (시니어 엔지니어 5년차, MSA 경험)

**[이야기]** 컨테이너 개념을 이미 알고 있어서 이야기 파트는 빠르게 읽음. 이야기 파트라고 불릴 만한 서사가 부족하고, 기술 문서에 가까운 흐름.

**[기술]** 이미지 레이어 설명이 없음. "이미지는 단순한 패키지가 아니라 레이어 구조다"라는 점이 빠져 있어서, 이후 Dockerfile을 다룰 때 레이어 캐시 개념이 갑자기 나오면 독자가 당황할 수 있음. 시니어 입장에서 개념 완성도가 약간 아쉬움.

**[실습]** `---` 수평선이 여러 군데에서 보임. `---\n\n---` 이중 사용은 편집 오류로 보임.

**[총평]** 기본 개념 설명은 탄탄하지만, 이미지 레이어 같은 심화 개념을 살짝 언급해 주면 더 완성도 있을 것.

**[점수]** 이야기 2/5 | 기술 4/5 | 실습 4/5

### 9. 강민서 (부트캠프 수료 1개월차, 명령어 처음)

**[이야기]** 말콤 맥린 이야기는 흥미로웠음. "내 컴퓨터에서 됐는데 서버에서 안 된다"는 게 어떤 상황인지 아직 경험이 없어서 와닿지는 않았음.

**[기술]** "가상화", "커널", "포그라운드", "백그라운드", "PID", "CMD" — 1챕터에 처음 듣는 용어가 너무 많음. 각각 정의가 달려 있지만 읽다 보면 앞의 정의를 잊어버림.

**[실습]** 첫 번째 실습(`docker run nginx`)에서 터미널이 멈추는 현상이 당황스러움. 사전 안내 없이 "포그라운드 상태"라고 사후에 설명되는 구조. 1.7.3에서 `coderyu5523/tomcat`을 그대로 따라 입력했더니 권한 오류 발생. 어디서 잘못된 건지 알 수 없어서 막힘.

**[총평]** 부트캠프 수료생 수준에서는 Linux 섹션과 생명주기 섹션을 합하면 분량이 지나치게 많음. 두 챕터로 분리하는 게 나을 수 있음.

**[점수]** 이야기 2/5 | 기술 2/5 | 실습 2/5

### 10. 윤채원 (CS 전공 학부 3년차, 이론 있음 실무 없음)

**[이야기]** 컨테이너 개념과 물류 역사 연결이 참신함. OS 이론을 알고 있어서 커널 설명이 편하게 읽힘. 이야기 서사가 없어서 교과서와 차이를 못 느낌.

**[기술]** 이미지 레이어 개념이 없음. CS 수업에서 레이어 기반 파일시스템(OverlayFS)을 배웠는데, 이 책에서 이미지를 "패키지"로만 설명하는 게 아쉬움. 나중에 Dockerfile에서 레이어 캐시가 나오면 연결이 안 될 것 같음.

**[실습]** `docker commit` 명령에서 `coderyu5523`은 저자 계정임을 바로 알아챘지만, 처음 독자는 모를 수 있음. `---` 수평선이 연속으로 두 번 나오는 부분은 편집 오류로 보임.

**[총평]** 기술적으로 정확하고 실습 흐름도 논리적. 이야기 요소가 더 있었으면 더 재미있게 읽었을 것.

**[점수]** 이야기 2/5 | 기술 4/5 | 실습 3/5
