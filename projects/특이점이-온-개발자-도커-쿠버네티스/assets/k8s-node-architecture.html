컴포넌트 위치 역할
API Server 컨트롤 플레인 모든 명령의 진입점. kubectl 명령이 여기로 들어온다
etcd 컨트롤 플레인 클러스터의 모든 상태 정보를 저장하는 저장소
Scheduler 컨트롤 플레인 Pod를 어느 노드에 배치할지 결정
Controller 컨트롤 플레인 원하는 상태와 실제 상태를 비교해서 자동 복구
kubelet 모든 워커 노드 컨트롤 플레인의 명령을 받아 컨테이너를 실행/관리
kube-proxy 모든 워커 노드 요청이 올바른 Pod로 전달되도록 네트워크 규칙 관리
Pod 워커 노드 컨테이너를 실행하는 가장 작은 단위
