## STEP 1: 씨앗

### Q1. 뭐에 대한 책이야?
도커+쿠버네티스를 예시를 통해 쉽게 이해하고 따라할 수 있는 개념서.

목차 구조:
- **1장. 왜 컨테이너인가**
  - 1.1 말콤 맥린 : 컨테이너의 아버지
  - 1.2 Docker와 Kubernetes
  - 1.3 이 책에서 만드는 것
- **2장. Docker 이해하기**
  - 2.1 Docker : 작동 원리
  - 2.2 Docker Desktop : 설치
  - 2.3 Docker CLI : 기본 명령어
  - 2.4 Linux : 컨테이너 안에서 쓰는 명령어
  - 2.5 컨테이너 : 생명주기와 옵션
  - 2.6 이미지 : 직접 만들어보기
  - 2.7 마운트 : 컨테이너에 저장소 연결하기
- **3장. Docker 다루기**
  - 3.1 프로비저닝(Provisioning)
  - 3.2 NGINX
  - 3.3 레디스(Redis) - 세션 저장소
  - 3.4 DB Server
  - 3.5 Docker Compose
  - 3.6 종합 실습 : 웹 사이트 만들기
- **4장. Kubernetes 시작하기**
  - 4.1 쿠버네티스(Kubernetes)
  - 4.2 미니큐브(Minikube)
  - 4.3 Deployment, ReplicaSet
  - 4.4 Service
  - 4.5 ConfigMap, Secret
  - 4.6 Volume
  - 4.7 미니큐브를 활용한 웹 사이트 만들기

### Q2. 왜 이 책을 쓰고 싶어?
도커 쿠버네티스를 처음 접하는 사람들에게 쉽고 재밌게 설명해주고 싶어서.

### Q3. 이 분야 경험이 어때?
도커로 개발할 수 있는 개발자. 아주 깊은 단계는 아님.

### Q4. 이 책에서 제일 하고 싶은 말?
"내 컴퓨터에서 되면 어디서든 된다" — 환경 차이 걱정을 없애는 게 도커의 핵심이라는 것.

### Q5. 독자가 뭘 이해하게 하고 싶어?
도커→Compose→K8s 흐름이 머릿속에 그려지는 것. 전체적인 그림을 이해하는 게 중요.

### Q6. 의도 밖 범위
- Spring Boot, Flask, Gradle 등 애플리케이션 코드는 깊게 다루지 않음
- Nginx 설정 심화 X
- K8s 운영/모니터링(Prometheus, HPA 등) X
- CI/CD 파이프라인 X
- Docker Swarm X
- 클라우드 배포(AWS EKS, GKE 등) X
- K8s 네트워크 심화(CNI, Ingress Controller) X
- 보안 심화(RBAC, Network Policy) X
- **포함**: 리눅스 기본 명령어 (명령어 실행에 필요하므로)

### 인사이트 답변
- **말콤 맥린 비유**: 1장에서 예시로 들고, 각 챕터에 전체를 관통하는 비유를 하나씩 배치
- **3.6 vs 4.7 웹사이트**: 같은 앱 대비 구조가 아님. 3장은 학습한 여러 컨테이너를 하나로 연결해서 배포하는 것이 목적
- **독자 수준**: 기본적인 프로그래밍은 할 수 있는 사람. 도커/쿠버네티스만 처음. 예시로 쉽게 풀되, 기반 지식(프로그래밍) 없이는 이해 불가
