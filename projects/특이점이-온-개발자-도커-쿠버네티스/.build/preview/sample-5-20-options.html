그림 5-20 배치 옵션 비교

현재 그림은 좌측에 Service, 우측에 Ingress가 있어 좌→우 읽기가 "Service 먼저"로 보입니다. 세 옵션 모두 Ingress가 먼저 읽히도록 조정한 안입니다.

(현재) 원본 — Service 좌상, Ingress 우하

좌→우 읽기 시 Service가 먼저 보임. 네트워크 흐름(Ingress → Service)과 반대 인상.

그림 5-20 (현재). Service 좌상 / Ingress 우하

옵션 A — 박스만 좌우 swap (송장 내부 유지)

Ingress를 좌상, Service를 우하로 옮김. 송장 내부 구조(위 주소 / 아래 바코드)는 유지. 화살표가 좌상→우하 대각선으로 교차하지만 우편 송장 실제 형태와 일치.

옵션 A. Ingress 좌상 → 바코드(아래), Service 우하 → 받는 곳 주소(위). 화살표가 대각선 교차.

옵션 B — 박스 swap + 송장 내부도 swap (시선 흐름 가장 매끄러움)

송장 내부 위쪽에 바코드, 아래쪽에 받는 곳 주소. Ingress 좌상 → 바코드(위) → 받는 곳 주소(아래) → Service 우하. 시선이 좌→우, 위→아래로 매끄러움. 단점: 실제 우편 송장은 보통 주소가 위라 비유의 직관성이 살짝 손상.

옵션 B. 송장 내부 위쪽=바코드, 아래쪽=받는 곳 주소. 좌→우, 위→아래 자연 흐름.

옵션 C — 좌우 평행 배치 (둘 다 같은 높이)

Ingress 좌측, Service 우측. 송장 가운데. 둘이 같은 높이로 좌→우 시선이 가장 명확하게 Ingress → 송장 → Service 흐름.

옵션 C. 둘 다 같은 높이. 좌→우 시선이 Ingress → 송장 → Service로 자연.
