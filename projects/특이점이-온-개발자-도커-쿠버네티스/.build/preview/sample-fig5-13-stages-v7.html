그림 5-13 — 단계별 샘플 v7 (K8s 실제 동작)

단계 1 · 사용자가 Ingress 리소스 적용
kubectl apply 한 YAML이 API Server를 거쳐 etcd에 저장됩니다
단계 2 · Ingress 컨트롤러가 변경을 감지하고 내부 설정 갱신
컨트롤러는 API Server에 watch를 걸어 두고 있어 변경이 즉시 전달됩니다
단계 3 · 외부 요청 처리
외부 요청이 오면 컨트롤러는 내부 설정으로 Service에 분기합니다 (etcd·API Server를 다시 거치지 않음)
그림 5-13. Ingress 리소스는 etcd에 저장되고, Ingress 컨트롤러가 API Server를 통해 미리 읽어 내부 설정을 갱신해 둡니다. 외부 요청은 그 내부 설정으로 즉시 분기됩니다
