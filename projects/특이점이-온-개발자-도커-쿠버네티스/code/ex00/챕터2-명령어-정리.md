## 2.4 Docker CLI 기본 명령어

### 이미지 다운로드

```bash
docker pull nginx
```

### 컨테이너 실행 (포그라운드)

```bash
docker run nginx
# CTRL + C로 빠져나오기
```

### 컨테이너 실행 (백그라운드)

```bash
docker run -d nginx
```

### 실행 중인 컨테이너 목록

```bash
docker ps
```

### 자주 사용하는 명령어

```bash
docker pull <이미지명>          # 이미지 다운로드
docker images                   # 로컬 이미지 목록 조회
docker logs <컨테이너ID>        # 컨테이너 로그 출력
docker ps -a                    # 종료된 컨테이너 포함 전체 목록
docker stop <컨테이너ID>        # 컨테이너 종료
docker rm <컨테이너ID>          # 종료된 컨테이너 삭제
docker rmi <이미지ID>           # 이미지 삭제 (-f로 강제 삭제)
```

## 2.5 Linux 명령어 (컨테이너 내부)

### Ubuntu 컨테이너 실행

```bash
docker run -it -p 80:80 ubuntu
```

### 탐색 명령어

```bash
pwd                # 현재 위치 경로 출력
cd <경로>          # 해당 폴더로 이동
cd ..              # 상위 폴더로 이동
ls                 # 파일/폴더 목록 출력
ls -l              # 상세 정보 출력
ls -a              # 숨김 파일 포함 출력
ls -la             # 숨김 파일 + 상세 출력
clear              # 터미널 화면 비우기
```

### 절대 경로 / 상대 경로

```bash
cd /home/ubuntu    # 절대 경로로 이동
cd /               # 루트 경로로 이동
```

### 파일/폴더 관리

```bash
mkdir <폴더명>           # 폴더 생성
touch <파일명>           # 빈 파일 생성
rm <파일명>              # 파일 삭제
rm -r <폴더명>           # 폴더 삭제 (하위 포함)
cp <원본> <사본>         # 파일 복사
mv <원본> <대상>         # 파일 이동 또는 이름 변경
```

### 패키지 관리 (apt)

```bash
apt update               # 패키지 목록 갱신
apt install -y nginx     # nginx 설치
nginx                    # nginx 실행
apt install -y net-tools # net-tools 설치
netstat -nlpt            # 포트 상태 확인
```

### vim 텍스트 편집

```bash
apt install -y vim       # vim 설치
vim test1.txt            # 파일 생성/편집
# i → 입력 모드 → 내용 작성 → ESC → :wq (저장 후 종료)
cat test1.txt            # 파일 내용 출력
```

### 프로세스 관리

```bash
ps -ef                       # 전체 프로세스 출력
ps -ef | grep <키워드>        # 특정 프로세스 검색
kill <PID>                    # 프로세스 안전 종료
kill -9 <PID>                 # 프로세스 강제 종료
```

### 파일 검색과 로그 확인

```bash
find / -name index.html       # 파일 이름으로 위치 검색
find / -name index*           # 패턴으로 검색
tail <파일>                    # 마지막 10줄 출력
tail -n 50 <파일>              # 마지막 50줄 출력
```

### 컨테이너 빠져나오기

```bash
exit                           # 컨테이너 종료 후 빠져나오기
# CTRL + P → CTRL + Q         # 컨테이너 유지하고 빠져나오기
```

## 2.6 컨테이너 생명주기

### exit vs detach

```bash
docker run -it --name dead ubuntu
exit
docker ps                          # 컨테이너 종료됨

docker run -it --name alive ubuntu
# CTRL + P → CTRL + Q
docker ps                          # 컨테이너 유지됨
```

### -d / -dit 옵션 비교

```bash
docker run -d nginx                # nginx: -d만으로 유지됨
docker ps

docker run -d ubuntu               # ubuntu: -d만으로는 즉시 종료
docker ps

docker run -dit ubuntu             # ubuntu: -dit로 백그라운드 유지
docker ps
```

### CMD 덮어쓰기

```bash
docker run -d ubuntu sleep 1000    # sleep이 메인 프로세스가 되어 유지
docker ps
```

### attach (메인 프로세스 접근)

```bash
docker run -dit ubuntu
docker ps
docker attach <컨테이너ID>
# CTRL + P → CTRL + Q로 빠져나오기
```

### exec (새 프로세스 접근)

```bash
docker exec -it <컨테이너ID> bash           # 새 bash 접속
docker exec <컨테이너ID> ps aux             # 프로세스 목록 확인
```

## 2.7 이미지 만들기

### Tomcat 실행

```bash
docker run -d -p 8080:8080 tomcat
docker ps
# 브라우저에서 localhost:8080 접속 → 404 에러
```

### Tomcat 컨테이너 수정

```bash
docker exec -it <컨테이너ID> bash
cd /usr/local/tomcat/webapps

mkdir ROOT
cd ROOT

apt update
apt install -y vim
vim index.html
# i → HTML 내용 작성 → ESC → :wq
# 브라우저에서 localhost:8080 접속 → 페이지 확인

exit   # exec 셸 종료 (컨테이너는 유지)
```

### docker commit (이미지 저장)

```bash
docker commit <컨테이너ID> <본인-dockerhub-id>/tomcat
```

### docker push (Docker Hub 업로드)

```bash
docker login
docker push <본인-dockerhub-id>/tomcat
```

## 2.8 마운트

### 바인드 마운트

```bash
# 호스트 폴더 생성
# Windows
mkdir -p /c/app/bind
# Mac / Linux
mkdir -p ~/app/bind

# 바인드 마운트 실행
# Windows
docker run -it --mount type=bind,src=C:/app/bind,dst=/app/bind ubuntu
# Mac / Linux
docker run -it --mount type=bind,src=$HOME/app/bind,dst=/app/bind ubuntu

# 컨테이너 내부
ls /app
touch /app/bind/a.txt
exit
```

### 볼륨 마운트

```bash
docker volume ls

docker run -it --mount type=volume,src=metacoding-volume,dst=/app/volume ubuntu
ls /app/volume
touch /app/volume/b.txt

exit
docker volume ls                   # 볼륨 유지 확인

# 새 컨테이너에서 기존 볼륨 데이터 확인
docker run -it --mount type=volume,src=metacoding-volume,dst=/app/volume ubuntu
ls /app/volume                     # b.txt 존재 확인
```
