# 씨앗

## 주제
MSA 입문 — 모놀리식에서 마이크로서비스로 전환하는 과정을 단계별로 학습하며 구조를 파악하는 책

## 집필 동기
기존 MSA 책이 너무 이론 중심이라 개념만 잔뜩 설명하고 실제로 만들어보는 과정이 부족하다. 코드로 직접 만들어보면서 MSA의 구조를 체감할 수 있는 책이 필요하다.

## 저자 경험
Spring Boot로 백엔드 개발 경험이 많고, MSA는 프로젝트 몇 개 해본 수준. 학습자 관점에서 MSA를 처음 접하는 사람이 어디서 막히는지 잘 안다.

## 핵심 메시지
"동기에서 비동기로, 단계별로 바꿔보면 MSA의 구조가 보인다"

## 책의 의도
MSA 전체 그림(서비스 분리 → 통신 → 배포 → 실시간 알림)을 하나의 쇼핑몰 프로젝트로 훑는다. 깊이 파기보다는 전체 조감도를 보여주는 것이 목적. 독자가 "MSA는 이런 구조로 돌아가는구나"를 이해하게 한다.

## 의도 밖 범위
- Spring Cloud (Eureka, Config Server, Circuit Breaker)
- Kafka 내부 동작 심화 (파티션, 리밸런싱, 오프셋)
- Kubernetes 운영 심화 (오토스케일링, Helm, 모니터링)
- 클린 아키텍처 심화 (헥사고날, DDD, CQRS)
- 테스트 전략 (단위, 통합, 컨트랙트 테스트)
- CI/CD 파이프라인
- 보안 심화 (OAuth2, mTLS)
- 모니터링/옵저버빌리티 (Prometheus, Grafana, 분산 트레이싱)
- 프론트엔드 심화 (React/Vue 등 프레임워크)
- 데이터베이스 심화 (샤딩, 이벤트 소싱)

※ 클린 아키텍처 기본 리팩토링과 WebSocket은 다룸

## 차별점
이론서가 아닌 실습서. 하나의 쇼핑몰 프로젝트가 동기 REST → 클린 아키텍처 + K8s 배포 → Kafka 비동기 사가 패턴 → WebSocket 실시간 알림으로 단계별 진화하면서, 독자가 각 단계에서 "왜 이렇게 바꿔야 하는지"를 직접 체감한다.
