CH03 한눈에 보기 — 응답 박스 통과 + 간격 넓힘

응답이 실제 K8s에서 요청 역순으로 박스(User → Gateway → Ingress → Client)를 통과하는 흐름으로 다시 그렸습니다. 요소 간 간격도 더 넓혀 라벨 공간을 확보했습니다.

옵션 A — 단계별 두 번 (응답 박스 통과)

Client·Ingress·Gateway 박스를 단계마다 두 번 배치. 요청은 위쪽 화살표 라인, 응답은 아래쪽 점선 라인으로 박스 역순 통과.

옵션 A — 단계별로 박스를 두 번 그려 흐름이 완전히 분리됨

옵션 B — K8s 진입점은 한 번 (응답 박스 통과)

Ingress·Gateway는 K8s 클러스터 안에 한 번만 그리고, Client만 두 번. 같은 박스를 두 단계의 요청·응답 화살표가 각각 통과.

옵션 B — Ingress·Gateway 한 인스턴스, 화살표만 두 번 통과

옵션 C — Ingress+Gateway 한 박스 (응답 박스 통과)

진입 경로를 "API 진입점" 한 박스로 단순화. 응답은 박스를 통과해서 Client로.

옵션 C — 진입점을 한 박스로 묶어 단순화
