옵션 C 시안 - 동기 REST vs 비동기 Kafka 대조 시퀀스

같은 시각 언어(시퀀스 다이어그램)로 두 그림을 그려, Order lifeline의 활성 박스 길이로 blocking 차이를 한눈에 보여줍니다.

Order 활성 (blocked·active) Orchestrator 활성 Product/Delivery 처리 점선 lifeline (대기)

그림 1-X · 동기 REST

그림 1-X. 동기 REST - Order가 응답까지 멈춤
시각 핵심: Order lifeline에 파란 활성 박스가 그림 끝까지 이어집니다(blocked). Product·Delivery 두 응답을 다 기다린 후에야 사용자에게 응답. blocking이 한눈에.

그림 1-Y · 비동기 Kafka

그림 1-Y. 비동기 Kafka - Order는 발행 즉시 자유
시각 핵심: Order lifeline의 파란 활성 박스가 그림 시작 직후 끝납니다(자유). 이후 흐름은 보라색 Orchestrator lifeline이 길게 받음. "발행하고 떠난다"가 한눈에.

두 그림이 대조되는 지점

관점동기 REST비동기 Kafka
Order 활성 박스길게 (전체 시간)짧게 (발행 직후 끝)
흐름 책임자Order가 직접 호출 사슬Orchestrator가 명령
참여자 수3 (Order·Product·Delivery)5 (Order·Kafka·Orchestrator·Product·Delivery)
응답 화살표점선 (요청 후 응답 대기)점선 (결과 이벤트, Order 아닌 Orchestrator로)

이 시안이 좋으면 챕터 1의 그림 1-8(가로 배치)·1-9(상하 배치)를 위 두 그림으로 교체. 같은 viewBox 폭으로 통일하여 인쇄 시 대조 가능.
