패키지 구조 비교 — 6가지 표현 시안

평면(챕터 2) → 책임별 분리(챕터 3) 변화를 표현하는 6가지 방법.

시안 1. 블록 박스 — "한 박스 → 네 박스"

패키지를 사각형으로. 한 덩어리가 네 조각으로 나뉘는 변환이 직설.

→
domain/

usecase/

web/
OrderController, DTO
시안 1. 블록 박스 — 평면 한 덩어리에서 네 책임별 조각으로

시안 2. 클린 아키텍처 4층 띠

엔티티 / UseCase / 어댑터 / 프레임워크 4층. 챕터 2는 다 섞여 있고, 챕터 3은 띠로 깔끔 분리.

챕터 3

domain/ — Order, OrderStatus
Use Cases
usecase/ — CreateOrderUseCase + 구현
Interface Adapters
web/ — Controller, DTO  |  repository/
Frameworks & Drivers
core/ — JWT, 예외  |  adapter/ — 외부 HTTP
