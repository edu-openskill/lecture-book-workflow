패키지 구조 비교 — 그림 3-5 시안

변경된 디렉토리만 인디고 악센트, 변하지 않은 것(core/·adapter/)은 회색 처리해서 변화가 즉시 보이게.

시안 A. 좌우 2단 (가장 단순)

챕터 2 (ex01)평면 구조
com/metacoding/order/
├── core/                # JWT·필터·예외
├── orders/             # 모든 게 한 패키지에
│   ├── Order.java
│   ├── OrderStatus.java
│   ├── OrderController.java
│   ├── OrderService.java
│   ├── OrderRepository.java
│   └── OrderRequest/Response.java
└── adapter/             # 외부 서비스 클라이언트
챕터 3 (ex02)책임별 분리
com/metacoding/order/
├── core/                # 그대로
├── domain/             # 엔티티 + 비즈니스 규칙
├── repository/         # Spring Data JPA
├── usecase/            # UseCase 인터페이스 + 구현
├── web/                # 컨트롤러 + DTO
└── adapter/             # 그대로
그림 3-5. 패키지 구조 비교 — 평면에서 책임별 분리로

시안 B. 좌우 + 화살표 (변화 방향 강조)

챕터 2평면 구조
com/metacoding/order/
├── core/
├── orders/
│   ├── Order.java
│   ├── OrderStatus.java
│   ├── OrderController.java
│   ├── OrderService.java
│   ├── OrderRepository.java
│   └── OrderRequest/Response.java
└── adapter/
→
챕터 3책임별 분리
com/metacoding/order/
├── core/
├── domain/
├── repository/
├── usecase/
├── web/
└── adapter/
그림 3-5. 패키지 구조 — 평면에서 책임별 분리로

시안 C. 위·아래 스택 (좁은 페이지에 안전)

챕터 2평면 구조 — 모든 게 한 패키지에
com/metacoding/order/
├── core/
├── orders/
│   ├── Order.java, OrderStatus.java
│   ├── OrderController.java, OrderService.java, OrderRepository.java
│   └── OrderRequest.java, OrderResponse.java
└── adapter/
챕터 3책임별 분리 — 4개 패키지로 나뉨
com/metacoding/order/
├── core/
├── domain/             # 엔티티 + 비즈니스 규칙
├── repository/         # Spring Data JPA
├── usecase/            # UseCase 인터페이스 + 구현
├── web/                # 컨트롤러 + DTO
└── adapter/
그림 3-5. 패키지 구조 비교
