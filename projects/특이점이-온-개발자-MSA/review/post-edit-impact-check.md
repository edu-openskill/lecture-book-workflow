### C-2. 2.6.5 단계 명세 ↔ 그림 2-2 ⑤ 어긋남 (中)

2.6.5 "보상 트랜잭션 설계" (L497~L510)의 단계 정의:

```
1단계: 재고 감소 (상품 서비스)
   └─ 실패 → 보상 없음 (아직 아무 일도 안 했으므로)
2단계: 배달 생성 (배달 서비스)
   └─ 실패 → 보상: 재고 복구 (1단계 되돌리기)
3단계: 주문 완료
   └─ 실패 → 보상: 배달 취소 + 재고 복구 (2, 1단계 되돌리기)
4단계: 성공 응답 반환
```

vs 그림 2-2 (L548~L578) 단계:
- ① 재고 차감 요청 / ② 차감 성공 응답 / ③ 배달 생성 요청 → 실패 / ④ 재고 복구 요청 (보상) / ⑤ **주문 트랜잭션 롤백**

vs 실제 코드 (L686~L734):
- try 안: 1) 주문 생성 → 2) 재고 차감 → 3) 배달 생성 → 4) 주문 완료
- catch 안: deliveryCreated 시 배달 취소 → productDecreased 시 재고 복구 → `throw new Exception500(...)` → 트랜잭션 롤백

세 곳의 단계 분류가 미세하게 다르다.
- 2.6.5는 "주문 완료" 자체가 실패할 수 있다고 가정하고 그 보상으로 "배달 취소 + 재고 복구"를 명시
