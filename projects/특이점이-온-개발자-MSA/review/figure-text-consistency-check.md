- **그림 1-10 (학습 흐름 카드)**: 4개 카드 모두 챕터별 실제 내용과 매칭. CH04 카드만 "Kafka + Saga"로 표기됐는데 CH04 본문은 "Orchestration Saga"임 → 의도된 추상화로 평가, 본문 핵심 정신과 모순되지 않음 (G-1 低)

- **그림 4-2 (메시지 큐 개념)**: SVG 라벨이 `PRODUCER: order-service` / `TOPIC: order-created` / `CONSUMER: orchestrator`로 구체 컴포넌트명을 박아둠. 그러나 캡션은 "메시지 큐 개념"으로 추상 — 즉, **그림은 구체화 / 캡션은 추상화** 사이 어색한 격차. 본문 4.2.1 헤딩 "토픽, 프로듀서, 컨슈머"가 추상 개념 설명 절이므로, 그림이 구체적 컴포넌트명을 미리 노출하는 점은 약간 빠른 도입 (D-1 中)

- **그림 4-7 / 4-8 / 4-9 (orchestrator 1~3단계)**: 본문 4.6.4 / 4.6.5 / 4.6.6 코드와 정합. 4.6.5 ②b "실패 → cancel-order-command (재고 복구 불필요)"는 본문 L699~L716 코드(차감 자체 실패 시 increase 호출 안 함)와 정확히 일치

  - **발견 없음** — 그림 4-14 통합 토폴로지는 본문 4.6·4.7과 정확히 매칭
