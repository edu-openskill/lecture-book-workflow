# 5.10 정리

Presigned URL 발급부터 S3 업로드, Lambda 리사이즈 처리, Spring 메타 저장, 목록/상세 조회까지 전체 흐름을 끝까지 진행했습니다. 업로드가 완료되면 실제 파일은 S3에 남고, 조회에 필요한 정보는 DB에 저장되는 구조가 정상적으로 동작하는 것을 확인했습니다. 

---

## **1) React 연동**

React 예제 프로젝트(react-presign-url)는 선택 과제입니다. 리액트까지 연결하면 브라우저에서 **업로드 → 미리보기 → 완료 저장 → 목록/상세 조회** 흐름을 화면에서 직접 확인할 수 있어 이해가 더 빠릅니다. 시간이 되는 경우에만 참고해서 실습을 확장해보면 됩니다.

---

### 실습코드

아래 spring과 react 서버의 깃헙 주소를 사용하여 진행합니다.

**Spring**

```java
https://github.com/metacoding-11-spring-reference/docker-presign-url
```

git clone 하고 .env 파일에 AWS IAM Access Key, Secret Key, S3 Bucket Name을 입력합니다.

[.env 파일 코드 수정하기]

```jsx
# ---------------------------

# ---------------------------
CLOUD_AWS_CREDENTIALS_ACCESS_KEY=YOUR_ACCESS_KEY
CLOUD_AWS_CREDENTIALS_SECRET_KEY=YOUR_SECRET_KEY
CLOUD_AWS_REGION=ap-northeast-2
CLOUD_AWS_S3_BUCKET=YOUR_BUCKET_NAME
```

 그리고 docker compose up으로 실행하여 spring 서버를 준비합니다.

## **2) 팁: CORS 설정**

React 개발 서버(예: http://localhost:5173)에서 S3 API를 호출하려면, S3 쪽에서 해당 Origin을 허용해야 합니다.

AWS S3 → 버킷 → 권한 탭 → CORS 설정으로 이동하여 아래 json을 입력합니다.

![image.png](5%2010%20%EC%A0%95%EB%A6%AC/image.png)

```java
[
    {
        "AllowedHeaders": [
            "*"
        ],
        "AllowedMethods": [
            "PUT",
            "GET",
            "HEAD"
        ],
        "AllowedOrigins": [
            "http://localhost:5173"
        ],
        "ExposeHeaders": []
    }
]
```

Presigned URL로 브라우저가 S3에 직접 PUT 업로드를 하려면, S3 버킷 CORS 규칙에서도 React 개발 서버 Origin을 허용해야 정상 업로드가 됩니다. 즉 “Spring API CORS”와 “S3 CORS”를 둘 다 맞춰야 브라우저에서 막히지 않습니다.

React와 Spring Presigned API를 연결해 업로드/미리보기/조회까지 흐름이 자연스럽게 이어지는지 확인해보면, 이 실습의 목적을 완전히 달성한 것입니다.
