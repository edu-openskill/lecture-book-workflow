## **2) 서비스 로직(완료 저장)**

**(입력) 경로: src/main/java/com/metacoding/spring_presign_url/image/ImageService.java**

```java
@Service
@RequiredArgsConstructor
public class ImageService {

    @Value("${cloud.aws.region}")
    private String region;
   
    /**
     * original/{uuid}.ext 업로드 이후 DB에 저장하는 로직
     */
    public ImageResponse.Detail checkAndSave(ImageRequest.completeRequest reqDTO) {

        String originalKey = reqDTO.key();

        // original/{uuid}.ext → uuid 추출
        String uuid = originalKey.replace("original/", "").split("\\.")[0];

        // resized/{uuid}.jpg 경로 구성
        String resizedKey = "resized/" + uuid + ".jpg";

        // AWS 정식 URL 구성
        String originalUrl = "https://" + bucket + ".s3." + region + ".amazonaws.com/" + originalKey;
        String resizedUrl = "https://" + bucket + ".s3." + region + ".amazonaws.com/" + resizedKey;

        ImageEntity entity = ImageEntity.builder()
                .uuid(uuid)
                .fileName(reqDTO.fileName())
                .originalUrl(originalUrl)
                .resizedUrl(resizedUrl)
                .createdAt(LocalDateTime.now())
                .build();

        imageRepository.save(entity);
        return ImageResponse.Detail.fromEntity(entity);
    }
}

```

완료 저장 단계에서는 Lambda가 로컬 서버로 웹훅요청을 할 수 없어,  서버가 이미 알고 있는 **S3 Object Key 규칙**을 그대로 활용합니다. 

클라이언트는 업로드가 끝난 원본 key(original/{uuid}.{ext})를 전달하고, 서버는 이 key에서 UUID를 추출한 뒤 리사이즈 결과 key를 resized/{uuid}.jpg 규칙으로 직접 조합하여 DB에 저장합니다.

## **3) 컨트롤러(완료 저장)**

**(확인) 경로: src/main/java/com/metacoding/spring_presign_url/image/ImageController.java**

```java
@RestController
@RequiredArgsConstructor
public class ImageController {

    @PostMapping("/complete")
    public ImageResponse.Detail complete(@RequestBody ImageRequest.completeRequest reqDTO) {
        return imageService.checkAndSave(reqDTO);
    }
}

```

완료 저장 요청을 받아 메타 정보를 저장합니다.
