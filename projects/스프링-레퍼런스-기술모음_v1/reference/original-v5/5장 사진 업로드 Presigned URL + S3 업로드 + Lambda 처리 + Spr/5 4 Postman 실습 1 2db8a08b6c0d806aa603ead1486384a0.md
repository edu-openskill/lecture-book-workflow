# 5.4 Postman 실습 1

Postman으로 presigned URL을 발급받고, 해당 URL로 S3에 PUT 업로드합니다. spring-presign-url-end터미널에 ./gradlew bootrun을 입력하여 서버를 실행합니다.

## **1) Presigned 발급 요청**

Postman을 실행하여 아래 요청 정보를 입력합니다.

![image.png](5%204%20Postman%20%EC%8B%A4%EC%8A%B5%201/image.png)

`POST` `http://localhost:8080/presigned`

Presigned 발급 요청 바디 예시입니다. 여기서 fileName의 확장자와 contentType의 확장자는 업로드 하고자 하는 이미지 파일과 동일해야 합니다. 예시) jpg = fileName.jpg, image/jpg

---

## **2) Presigned 응답 확인**

응답에서 presignedUrl, key를 확인합니다.

![image.png](5%204%20Postman%20%EC%8B%A4%EC%8A%B5%201/image%201.png)

응답 받은 presignedUrl로 이미지를 업로드 하기 위해 **클릭하여 새창으로 이동**합니다. 또는 복사하여 포스트맨의 새창 Url 칸에 붙여넣기 하세요. 

---

## **3) S3 PUT 업로드**

![image.png](5%204%20Postman%20%EC%8B%A4%EC%8A%B5%201/image%202.png)

HTTP 요청을 PUT으로 변경하고, Header 탭에서 Content-Type : image/png 를 추가 합니다.

---

![image.png](5%204%20Postman%20%EC%8B%A4%EC%8A%B5%201/image%203.png)

body 탭에서 binay를 선택한 후 원하는 파일을 선택합니다. 여기서 presignedUrl 발급요청시 넣은 content-Type과 동일한 확장자를 파일을 업로드 해야합니다.

설정 후 Send를 눌러 업로드는 요청합니다.

![image.png](5%204%20Postman%20%EC%8B%A4%EC%8A%B5%201/image%204.png)

응답 200을 확인 후 AWS S3 metacoding-base64/original 폴더를 확인합니다.

---

## **4) S3 업로드 확인**

![image.png](5%204%20Postman%20%EC%8B%A4%EC%8A%B5%201/image%205.png)

S3 콘솔에서 `original/` 경로에 업로드된 객체를 확인합니다. 

![image.png](5%204%20Postman%20%EC%8B%A4%EC%8A%B5%201/image%206.png)

이미지 확인을 위해 클릭하고 열기 버튼을 클릭하여 업로드한 이미지와 동일한지 확인합니다.
