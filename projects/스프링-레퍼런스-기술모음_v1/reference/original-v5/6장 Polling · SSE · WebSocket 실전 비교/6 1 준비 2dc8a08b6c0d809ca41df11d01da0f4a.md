# 6.1 준비

### **Polling 개념과 특징**

Polling은 클라이언트가 일정 주기마다 서버에 “변경됐어?”를 물어보는 방식입니다. 구현이 단순하고 HTTP 기반이라 어디서든 동작하지만, 변화가 없어도 계속 요청이 발생해 트래픽이 늘고 “최대 지연 = 폴링 주기”라는 한계가 생깁니다.

### **SSE(Server-Sent Events) 개념과 특징**

SSE는 클라이언트가 서버에 한 번 연결을 열어두면, 서버가 이벤트를 계속 밀어주는 “단방향 스트림”입니다. 브라우저 기본 API(EventSource)가 있어 프론트가 가볍고, 텍스트 기반 이벤트 전송에 강합니다. 

### **WebSocket 개념과 특징**

WebSocket은 한 번 연결을 맺으면 양방향으로 메시지를 주고받는 방식입니다. 채팅, 협업 편집, 실시간 제어처럼 상호작용이 많을수록 유리합니다. 대신 연결 상태(세션), 재연결, 인증/권한, 메시지 라우팅 등을 앱 차원에서 관리해야 합니다. 

### **한눈에 보는 차이 표**

| **구분** | **Polling** | **SSE** | **WebSocket(STOMP)** |
| --- | --- | --- | --- |
| 통신 방향 | 요청/응답 반복 | 서버 → 클라(단방향) | 양방향 |
| 연결 | 매 요청마다 | 1회 연결 유지 | 1회 연결 유지 |
| 지연 | 폴링 주기 영향 | 거의 실시간 | 실시간 |
| 구현 난이도 | 낮음 | 중간 | 높음 |
| 적합한 케이스 | 완료 여부 확인, 간단 상태 | 알림/진행률/스트림 이벤트 | 채팅/협업/양방향 제어 |
