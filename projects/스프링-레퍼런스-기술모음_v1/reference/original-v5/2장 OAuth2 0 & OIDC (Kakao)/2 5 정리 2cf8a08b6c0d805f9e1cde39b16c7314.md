# 2.5 정리

Authorization Code, Credential 방식과 OIDC(OpenID Connect)를 정리하며 마무리하는 챕터입니다.

---

## **1) Code(SSR) vs Code + OIDC(REST)**

### **1. Code 방식 (SSR 중심)**

![image.png](2%205%20%EC%A0%95%EB%A6%AC/image.png)

Code 방식은 웹 브라우저 환경에서 주로 사용되던 로그인 방식입니다. 사용자가 로그인하면 인가 코드(code)를 카카오에 등록된 redirect URI로 콜백받고, 서버는 이 code로 Access Token을 발급받습니다. 이후 해당 Access Token을 사용해 **카카오 서버에 사용자 정보를 직접 조회**하여 로그인 처리를 완료합니다.

---

### **2. Code + OIDC(REST)**

![image.png](2%205%20%EC%A0%95%EB%A6%AC/image%201.png)

Code + OIDC 방식은 인가 코드로 토큰을 요청할 때 **ID Token(OIDC 토큰)** 을 함께 발급받는 방식입니다. 서버는 카카오 서버에 사용자 정보를 다시 요청하지 않고, 카카오가 제공하는 **공개키(JWKS)** 를 조회해 ID Token의 서명을 검증한 뒤, **id_token** 안에 포함된 사용자 정보를 그대로 사용합니다.

**카카오 응답 토큰 예시**

```dart

  "expires_in": 21599,
  "refresh_token": "ZDFhMzQ1YjctY2QxMi00NjQyLWE3MzEtYmFjY2QxMjM0NTY3",
  "refresh_token_expires_in": 5183999,
  "id_token": "eyJhbGciOiJSUzI1NiIsImtpZCI6IjE2Yi1rZXktaWQiLCJ0eXAiOiJKV1QifQ.eyJpc3MiOiJodHRwczovL2thdXRoLmtha2FvLmNvbSIsImF1ZCI6IjEwMTIzNDU2Nzg5MCIsInN1YiI6IjM0NTY3ODkwMTIzNDU2NyIsImV4cCI6MTczNDk4ODAwMCwiaWF0IjoxNzM0OTg0NDAwLCJub25jZSI6Im5vbmNlLTEyMyJ9.SIGNATURE_BYTES"

```

## 2) Code vs Credential

![image.png](2%205%20%EC%A0%95%EB%A6%AC/image%202.png)

**Code 방식**은 브라우저 환경을 전제로 한 로그인 구조로, 인가 코드를 서버가 받아 토큰을 발급하고 사용자 정보를 조회하는 서버 중심 방식입니다. 

반면 **Credential 방식**은 모바일 앱처럼 신뢰 가능한 환경에서 SDK가 로그인과 사용자 확인을 모두 처리하며, 서버는 전달받은 토큰의 유효성만 검증하는 역할을 합니다.
