## 2) Oauth에 대해서

> **Oauth란** **사용자가 신뢰할 수 있는 플랫폼(카카오)을 통해, 제3자(우리 서비스)에게 자신의 데이터 접근 권한을 안전하게 맡기는 표준 프로토콜 입니다.**
> 

아래 그림은 OAuth의 권한 위임 개념을 **호텔 예시**로 표현한 것입니다

![image.png](2%201%20%EC%A4%80%EB%B9%84/image%2011.png)

사용자는 청소 업체에게 청소 권한만 줄 수 있게 호텔 프런트에게 요청하며 호텔 프런트는 청소업체에게 청소만할 수 권한인 카드키를 발급해줍니다. Oauth도 동일하게 데이터에 접근할 수 있는 권한을 위임받아 사용하는 프로토콜입니다.

### **OAuth 2.0의 두 가지 인증 방식 – Code vs Credential**

OAuth 2.0에는 대표적으로 **code 방식**과 **credential(클라이언트/SDK 중심) 방식** 흐름이 있습니다.

**Code 방식**

사용자가 브라우저에서 로그인하면 **임시 코드(Code)**를 받고, 서버가 이 코드를 사용해 **Access Token** 을 요청받는 구조입니다. REST API를 사용하여 요청합니다.

**Credential 방식**

모바일 앱(Android/iOS), React 등에서 사용하는 방식입니다. 카카오 SDK 같은 것을 설치하면 로그인부터 토큰 발급까지 **클라이언트에서 직접 처리** 합니다.

이 책은 **서버에서 인증 흐름을 명확히 이해하기 위해 Code 방식을 사용**합니다.
