# 7.4 FFmpeg 인코딩 & HLS 조각화 구현

### **1. FFmpeg 실행 및 HLS 출력 생성**

**(확인) 경로: rtmp/src/main/java/com/metacoding/hls/service/HlsService.java**

```java
FFmpegBuilder builder720 = new FFmpegBuilder()
        .setInput(inputPath)
        .addOutput(output720)
        .addExtraArgs("-b:v", "2500k")
        .addExtraArgs("-maxrate", "2500k")
        .addExtraArgs("-bufsize", "5000k")
        .addExtraArgs("-vf", "scale=-2:720")
        .addExtraArgs("-hls_time", "10")
        .addExtraArgs("-hls_list_size", "0")
        .addExtraArgs("-f", "hls")
        .done();

```

처음 보면 옵션이 많아 보이지만, 핵심은 아래 5가지입니다.

- `vf scale=-2:720`: 세로를 720으로 맞추고, 가로는 비율을 유지합니다.
- `b:v`, `maxrate`, `bufsize`: 비트레이트 품질과 변동 폭을 조절합니다.
- `hls_time 10`: **10초 길이의 조각(ts)**을 만듭니다.
- `hls_list_size 0`: 모든 조각을 m3u8 목록에 담습니다.
- `f hls`: 출력 포맷을 HLS로 지정합니다.

즉, 이 설정은 **원본 영상 → 10초 단위 ts 조각 + m3u8 목록**을 만드는 역할입니다.

<aside>
💡

HLS 란?

HLS는 **영상을 인터넷에서 끊기지 않게 보기 위해 “잘게 쪼개서” 보내는 스트리밍 방식입니다.**

- 영상 1개(mp4)를 그대로 보내는 게 아니라
    
    **10초 같은 짧은 조각 파일(.ts)** 여러 개로 나눠서 보냄
    
- 그리고 “조각 목록”을 담은 **재생 목록 파일(.m3u8)** 을 같이 만듦
- 플레이어는 먼저 .m3u8을 받고, 거기에 적힌 .ts 조각들을 순서대로 계속 받아서 재생함

즉, **HLS = (m3u8 목록) + (ts 조각들)** 로 돌아가는 스트리밍 방식

</aside>

### **2. 저장 경로/파일명 규칙**

**(확인) 경로: rtmp/src/main/java/com/metacoding/hls/service/HlsService.java**

```java
String baseName = "video";
String output720 = new File(CONVERT_DIR_720, baseName + ".m3u8").getAbsolutePath();
String output1080 = new File(CONVERT_DIR_1080, baseName + ".m3u8").getAbsolutePath();

```

`baseName`이 `video`로 고정되어 있으므로 결과는 항상 이렇게 만들어집니다.

- `upload/hls/720p/video.m3u8`
- `upload/hls/720p/video0.ts`, `video1.ts`, ...
- `upload/hls/1080p/video.m3u8`

이 방식은 “최신 영상 하나만 재생”하는 실습에 적합합니다. 여러 파일을 동시에 서비스하려면 `baseName`을 업로드 파일명으로 바꾸는 방식이 필요합니다.
