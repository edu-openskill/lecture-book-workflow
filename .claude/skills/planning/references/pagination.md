# 분량 관리 규칙

> 출처: v1 planning/pagination + v3 적응
> 관련 스킬: D6.분량-계산기

---

## 절대 제약

| 항목 | 기준 |
|------|------|
| 전체 분량 | **150페이지 권장** (초과 허용, 저자 리뷰 시 빼기 작업) |
| 단일 챕터 | **20페이지 이하** |
| 1페이지 | ≈ 500자 (한글 기준) |
| 이미지 | **필요한 곳마다** (비유·핵심 개념·수식/그래프). 장수 쿼터 아님 — 글로 충분하면 생략, 도움 되면 충분히 |

---

## 페이지 추정 기준

| 요소 | 예상 분량 |
|------|----------|
| 마크다운 40줄 | ≈ 1페이지 |
| Mermaid 다이어그램 | ≈ 0.5페이지 |
| 이미지 | ≈ 0.5~1페이지 |
| 코드 블록 (10줄) | ≈ 0.3페이지 |

---

## 페이지 배분 매트릭스

```
[Part 1] Ch.1 (Xp) / Ch.2 (Xp) / ... → 소계: Xp
[Part 2] Ch.3 (Xp) / Ch.4 (Xp) / ... → 소계: Xp
─────────────────────────────────────────
총계: ≤ 150p (권장)
```

---

## 챕터 내부 구성 비율 (v3 스토리텔링)

| 파트 | 비율 |
|------|------|
| 이야기 파트 — 도입 | 10% |
| 이야기 파트 — 개념(비유) | 20% |
| 이야기 파트 — 시나리오 진행 | 40% |
| 이야기 파트 — 이것만은 기억하자 | 5% |
| 기술 파트 — 용어 정리 + 코드 | 20% |
| 기술 파트 — 더 알아보기 | 5% |

> v3 비율: 이야기:기술 ≈ 75:25 (스토리텔링 중심)

---

## 분량 검증 체크리스트

```
□ 챕터 수 × 평균 페이지 ≤ 150p?
□ 비유·핵심 수식마다 그림이 있나(필요한 곳 기준, 장수 아님)?
□ 자연스러운 학습 곡선?
□ 각 챕터가 독립적으로 의미 있는가?
□ 첫 챕터에서 동작하는 결과물이 있는가?
□ 20p 넘는 챕터가 있으면 분할 검토
```
