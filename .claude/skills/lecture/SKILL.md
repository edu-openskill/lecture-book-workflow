# 강의 해부 스킬

강의 트랙에서 `code` 스킬에 대응한다(완성 코드 분석 대신 강의 해부).
입력이 코드가 아니라 **강의 자료(판서 대본 + 자막)** 이므로, "코드 구조"가 아니라 **"개념 구조"** 를 뜯는다.

## 로드 시점

| STEP | 상황 |
|------|------|
| STEP 2. 강의 해부 | 강별 판서대본·자막에서 개념/비유/선수관계/오개념 추출 |
| STEP 4. 뼈대 (보조) | 추출한 개념을 챕터 목차에 배치할 때 참조 |

## 입력

- `sources/NN강/판서대본.md` — 1차 소스(필수)
- `sources/NN강/caption-clean.md` — 2차 소스(있으면)
- `planning/seed.md` — 의도 필터

## 담당 스킬 (원자적 도구 4종)

| 스킬 | 입력 → 출력 | 핵심 규칙 |
|------|------------|----------|
| L1. 개념-추출기 | 판서대본 → 강별 핵심 개념 목록 | 슬라이드 제목·"개념 정의" 블록을 우선 추출. 한 개념 = (이름 + 한 줄 정의 + 등장 슬라이드) |
| L2. 비유-인벤토리 | 판서대본·자막 → 비유 목록 | 강사가 쓴 비유를 그대로 수집(전학생=어텐션, 보따리=컨텍스트 등). **새 비유를 지어내지 않는다.** 원문 비유가 곧 책의 비유 |
| L3. 선수편-매퍼 | 목차 메타 + 개념 → 강 간 의존성 | "A강을 이해하려면 B강 개념 필요" 그래프. 목차의 `선수` 메타를 1차 근거로 |
| L4. 오개념-수집기 | 판서대본·목차 → 강별 오개념 | 강이 부수려는 "흔한 착각". 책의 이야기 파트 도입(문제 제기)으로 직결 |

> 비유는 **만들지 말고 수집한다.** 강의에서 이미 검증된 비유를 책이 물려받는 것이 이 하네스의 핵심.
> (원본 `writing/`의 C1.비유-생성기는 대본에 비유가 없을 때만 보조로 쓴다.)

## 흐름

```
--- AI 자동 분석 (강별 반복) ---
1. L1.개념-추출기   -> 강별 핵심 개념 목록
2. L2.비유-인벤토리 -> 강사 비유 수집
3. L4.오개념-수집기 -> 강별 오개념
--- 전체 종합 ---
4. L3.선수편-매퍼   -> 강 간 의존성 그래프
--- 의도 필터 ---
5. planning/seed.md 의도 필터 적용 -> 책에 담을 개념 / 뺄 개념 분류
    |
[인사이트 검토] -> 강의엔 있으나 책엔 빠진 연결고리 질문
    |
[감수 검토] (기술 감수자 + 독자 대변인)
    |
저자 확인 -> planning/lecture-analysis.md 생성
```
