## 다이어그램 유형 선택

| 유형 | 도구 | 사용 시점 |
|------|------|----------|
| 단순 흐름도 (≤7노드) | Mermaid | 간단한 구조, 빠른 작성 |

| 시퀀스/단계 흐름 | **플로우 카드** | ReAct, 파이프라인, 요청-응답 흐름 |

## 플로우 카드 실행 흐름 (메인 세션)

서브에이전트는 이미지를 유저에게 표시할 수 없으므로, 플로우 카드는 **메인 세션에서 직접 실행**한다.

1. 챕터 컨텍스트 읽기 (scenario, outline, 위아래 섹션)
2. 유저에게 방향(가로/세로) 입력 받기
3. 다이어그램 내용 확정 (flow 명세 기반)
4. 4가지 레이아웃 변형 HTML 생성 → 브라우저에서 표시
5. 유저 선택 → PNG 렌더링 → `assets/CH{N}/diagram/` 저장
6. 불만족 시 수정 루프 (이전 버전으로 되돌리기 가능)

## 핵심 규칙

### Mermaid
- **`flowchart`** 키워드 (`graph` 아님)
- 노드: **최대 7개** — 넘으면 subgraph로 분리
- 특수문자 라벨: **큰따옴표 필수**
- 엣지 라벨: `A -- "텍스트" --> B` 형식
- **Bold/Italic (`**`, `*`): Mermaid 안에서 절대 금지**

### 이미지
- 이미지 삽입: `![](경로)` — alt 텍스트 비움 (캡션은 이탤릭으로 별도 작성)
- 캡션: 이미지 바로 아래에 `*설명*`만 작성. `그림 N-N:` 접두어 금지 (Typst가 자동 부여)
- 집필 시 **플레이스홀더 삽입** → 실제 이미지는 집필 완료 후 생성
- 플레이스홀더 2종: `[GEMINI PROMPT: ...]` (개념도) / `[CAPTURE NEEDED: ...]` (실습 캡처)
- 이미지 경로: `assets/CH{N}/` 챕터별 폴더
- **스크린샷과 코드 블록 중복 금지**: 스크린샷이 있으면 출력 코드 블록 제거
