## 핵심 규칙

### Mermaid
- **`flowchart`** 키워드 (`graph` 아님)
- 노드: **최대 7개** — 넘으면 subgraph로 분리
- 특수문자 라벨: **큰따옴표 필수**
- 엣지 라벨: `A -- "텍스트" --> B` 형식
- **Bold/Italic (`**`, `*`): Mermaid 안에서 절대 금지**

### 이미지
- 캡션: `*그림 {장}-{순번}: 설명*` 형식
- 집필 시 **플레이스홀더 삽입** → 실제 이미지는 집필 완료 후 생성
- 플레이스홀더 3종: `[IMAGE PROMPT: ...]` (개념·비유, Codex/GPT 생성) / `[PLOT SCRIPT: ...]` (정확한 수식·좌표 그래프, matplotlib 실행) / `[CAPTURE NEEDED: ...]` (실습 캡처) — 반드시 `references/image.md`의 블록 형식으로 작성(맨 한 줄 표기는 감지 안 됨)
- 이미지 경로: `assets/CH{N}/` 챕터별 폴더
- **스크린샷과 코드 블록 중복 금지**: 스크린샷이 있으면 출력 코드 블록 제거
- **멀티패널(한 장으로 안 되면 2~3장)**: 한 장에 단계·비교·과정을 다 욱여넣지 말 것. 단계가 여럿이거나(예: 경사하강 1걸음→2걸음→수렴), 비교가 필요하면(작은 보폭 vs 큰 보폭) **연속 2~3장으로 쪼개** 가로(나란히)·세로(위→아래) 배치한다. 상세: `references/image.md` §6.
- ⚖️ **검산되면 matplotlib, 아니면 생성형**: 그림이 축·좌표·수식·명명된 점처럼 **독자가 검산할 수 있는 주장**을 하면 `[PLOT SCRIPT]`(matplotlib)로 그린다 — 생성형은 곡선 연속성·점 위치를 **원리적으로 보장 못 함**(프롬프트 강도로 해결 불가, Ch.8 ex1이 깨진 이유). 정성 라벨뿐인 개념 스케치·순수 비유 삽화만 `[IMAGE PROMPT]`. 기준표: `references/image.md` §0.

### 이미지 (필요한 곳에 넣는다 — 장수 쿼터 아님)
이미지는 **정해진 장수를 채우려고** 넣는 게 아니라 **이해를 돕는 자리에** 넣는다. 다만 글만 길게 이어져 단조롭지 않게 충분히 넣는다(경계로 삼을 것은 "그림이 너무 적은 것"이지 "특정 장수").

**넣는 자리**
- 이야기 파트: 새 **비유**가 나오는 곳(도입 비유·핵심 비유). 비유를 그림으로 받치면 몰입이 오른다. *(이야기 파트는 코드만 금지, 그림은 권장)*
- 기술 파트: **직관이 필요한 핵심 개념·수식/그래프**(곡선·기하·구조 등). 수식을 직관으로 받칠 그림.

**넣지 않는 자리**: 글로 충분히 전달되는 단순 서술, 이미 다른 그림이 같은 내용을 보인 곳. 장수를 맞추려는 빈자리 채우기 금지.

- 도구: **흐름·구조·관계** = Mermaid, **비유·정성 개념** = `[IMAGE PROMPT]`(Codex/GPT), **정확한 수식·좌표 그래프** = `[PLOT SCRIPT]`(matplotlib), **실습 결과** = `[CAPTURE NEEDED]`.
- 결과적으로 개념 챕터는 보통 여러 장이 되지만 **숫자를 목표로 삼지 말 것**. 분량이 밀리면 덜 필요한 그림(이야기 컷)부터 줄인다.
