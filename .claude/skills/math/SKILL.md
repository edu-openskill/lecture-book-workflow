# 수식·예제 스킬 (개념 트랙)

## 로드 시점
- STEP 5 챕터 집필(개념 트랙) 기술 파트
- STEP 3 계산예제 설계

## 핵심 규칙

### ★ 비유→수식→직접풀이 3단 (매 강, 예외 없음)
> 이 책의 정체성: "필요한 만큼만, 대신 **직접 풀어서** 확실히." 비유로 끝내지 않는다.
- 모든 핵심 개념은 **① 비유(직관) → ② 수식(LaTeX 정의) → ③ 직접 풀이(구체 수치 worked example)** 3단을 모두 거친다. 셋 중 하나라도 빠지면 미완성.
- 비유만으로 끝내지 마라. 비유는 입구일 뿐, 반드시 수식과 손풀이로 닫는다.

### ★ 유도 중간 생략 금지
- "이렇게 됐으니 다음은 이렇게 되고, 그다음은 이렇다" — **한 줄= 한 단계**로 끝까지 잇는다.
- **단언 금지**: "상수는 ln a이다", "d/dx ln x = 1/x이다" 처럼 결과만 던지지 않는다. *왜* 그렇게 되는지 유도 과정을 깐다. "독자가 이미 안다"고 가정하고 중간을 건너뛰지 않는다.
- 새 기호는 산문으로 단언하지 말고 **비유 → 손계산 한 줄 → 정의** 순으로 깐다(첫 접촉의 밀도가 이해를 가른다).

### ★ 새 개념·어려운 주제는 집필 전 조사 먼저
- 경사하강·최적화처럼 갈래가 많은 주제는 **집필 전에** 인터넷·논문을 조사해 `planning/research-*.md`로 남기고, 무엇을 본문/더알아보기/범위밖으로 둘지 스코프를 먼저 확정한다(예: `research-경사하강-방법론.md`).
- 오늘 날짜 기준으로 사실을 검증하고, 더 쉽게 설명한 글이 있으면 참고한다([사실검증] 체크).

### ★ 선수 지식 대조 (known.md)
- 집필 전 `planning/known.md`로 독자가 아는 개념·기호를 확인한다. 원장에 없는 용어를 쓰려면 **이 챕터 안에서** 처음부터 설명하고, outline상 뒤 챕터 개념은 당겨쓰지 않는다(자세한 절차는 `workflow/review-guide.md` 2.5).

### 수식 표기
- **LaTeX만 사용**. 인라인 `$...$`, 블록 `$$...$$`. Typst math 직접 작성 금지(빌드 시 Pandoc이 변환).
- 정의는 비유 → 정식 정의 순서. 기호는 처음 등장 시 한 번 풀어 설명.

### 계산 예제(worked example)
- 반드시 **구체적 수치**로 시작. 추상 일반식만 두지 않는다.
- 단계마다 한 줄 = 한 연산. 각 단계에 수식 + 한 줄 설명.
- 끝에 "왜 이게 AI에 쓰이나" 한 줄.

### 그래프는 `[PLOT SCRIPT]`(matplotlib)로 — 생성형 금지
- 함수 곡선·곡선 위의 점·화살표 주석·등고선처럼 **좌표·수식이 정확해야 하는 그래프**는 생성형 이미지(`[IMAGE PROMPT]`)로 그리지 않는다. 생성형은 곡선 연속성·점 위치를 **원리적으로 보장 못 함**(Ch.8 ex1: 포물선이 조각조각 끊겼던 사고).
- 점의 y값은 손으로 적지 말고 `f(x)`로 **계산**한다(코드는 안 틀린다). 형식·러너: `visual/references/image.md` §0·방식 C + `image-gen` 스킬 `plot_gen.py`.
