# 메모 박스 (:::memo)

> 캐릭터가 수첩·포스트잇·쪽지에 **관찰, 생각, 꼭 기억해야 할 내용**을 적는 장면을 시각화하는 directive 박스. 이야기 파트의 감정을 끊지 않으면서 요점을 "메모하듯이" 남기는 용도.

## 언제 쓰는가 (자율 호출 트리거)

다음 상황 중 **하나라도** 해당하면 본 컴포넌트를 사용한다.

| 트리거 | 예시 |
|--------|------|
| 실험 결과를 정리 | OCR 결과를 수첩에 적는 장면 (CH10 §10.2) |
| 시행착오 기록 | "두 번째 시도에서도 실패. 원인은…" |
| 문제를 좁히는 생각 | 화이트보드 앞에 서기 전 수첩에 문제 정리 (CH08 §8.2, CH09 §9.1) |
| 꼭 기억할 규칙·발견 | "이 구조는 나중에 또 쓰일 것 같다" |

**쓰지 않는 경우**: 챕터 닫기 총정리는 `:::remember`, 용어 확정은 `:::term-box`, 보조 정보는 `:::tip`. 메모는 **이야기 안에서 캐릭터가 직접 쓰는 메모장** 톤이라는 점이 다르다.

## 사용 챕터

- CH06 §6.6 (정형·비정형·복합 질문 처리 경로 정리)
- CH08 §8.2 (세 문제 정리)
- CH09 §9.1 (질문 이해 문제 정리)
- CH10 §10.2 (OCR 실험 결과 정리)

## Markdown 사용 예

```markdown
:::memo
**— 실험 메모 —**

1. **한글 오인식**
   - '정보보안 서약서' → '스보보안 서의서'
   - 도장·고유명사 근처 글자가 뭉개짐
2. **표 구조 소실**
   - 행·열이 사라지고 한 줄로 늘어섬
   - "김철수 대리 인사팀 박영희 과장 개발팀"
   - 누가 어느 부서인지 추론 불가
:::
```

**렌더 CSS**: `components.css:117-149` `.memo-box` (아이보리 배경 `#fdfaf2` + 바랜 노트 테두리 `#e6dfc6`, 모노스페이스 폰트, 2px 포스트잇 그림자). `.memo-box > p:first-child`는 `— 제목 —` 라벨 스타일(subtle 컬러 + letter-spacing 2px). `.memo-box ol > li`는 번호 매김 + 굵은 제목. 중첩 `ul`은 `·` 마커로 서브 항목.

## 제목 톤 (변형)

| 변형 | 언제 | 첫 줄 패턴 |
|------|------|-----------|
| **실험 메모** | 실험·관찰 정리 | `**— 실험 메모 —**` |
| **문제 정리** | 깨달음·문제 식별 | `**— 문제 정리 —**` |
| **생각 정리** | 내면 사색 정리 | `**— 생각 정리 —**` |
| **기억할 것** | 규칙·발견 | `**— 기억할 것 —**` |
| **쪽지** | 단일 발견 한 줄 | `**— 쪽지 —**` |

## 삽입 규칙

1. **캐릭터의 물리적 행동이 선행**한다. 예: "수첩을 펼쳤습니다", "포스트잇을 꺼냈습니다", "서랍에서 수첩을 꺼냈습니다"
2. 첫 줄은 `**— 제목 —**` 패턴
3. 항목은 1~4개. 5개 이상이면 수첩 느낌이 사라진다
4. 숫자 번호 매김 `1. **제목**` + 서브 불릿 `- 상세`
5. 마지막에 `**→ 결론**`을 추가해도 좋다 (선택)

## 피해야 할 것

- **본문 해설의 대체로 쓰지 말 것**. 메모는 캐릭터가 "정리하는 장면"의 소품. 기술 해설이 필요하면 본문 문단으로.
- **항목 수 5개 초과 금지**. 수첩 한 페이지 느낌을 유지하려면 1~4개가 적정.
- **캐릭터 행동 없이 단독 삽입 금지**. 반드시 "수첩을 펼쳤습니다" 같은 행동 문장이 선행.
- **코드블록·이미지·표 내부 삽입 금지**. 텍스트만.
- **챕터 닫기에 사용 금지**. 그 자리는 `:::remember`.
- **같은 챕터에서 3개 초과 사용 금지**. 남발하면 "수첩 장면"의 힘이 사라진다.
