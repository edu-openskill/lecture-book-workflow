# 판서 대본 작성 가이드

슬라이드는 뼈대, **대본이 진짜 강의**다. "PPT 읽는 사람"이 되지 않기 위한 문서.
강의는 슬라이드로 진행하고, 시뮬레이터(있다면)는 강의 끝에 복습용으로 켠다.
대본은 슬라이드별로 **무엇을 말할지**에 집중한다.

## 형식

각 슬라이드는 **대본(무엇을 말할지)**으로 쓰고, 매핑되는 개념이 있으면 그 밑에 **개념 정의**를 둔다.
**시간 표기·이모지·"무엇을 그려라" 같은 연출 지시는 넣지 않는다.**

- **대본** — 그 슬라이드에서 실제로 할 말(구어체, 전학생에게 설명하듯). 하나의 이야기로 흐르게.
- **개념** (매핑되는 개념이 있을 때만) — 정확한 정의 + 비유와의 연결. 필요하면 한 줄 참고(책·논문).

```
## 슬라이드 N — 제목
대본:
(실제로 할 말…)
개념 — 개념명:
(정확한 정의 + 비유 매핑)
참고: (출처, 있을 때만)
```

**오프닝(슬라이드 1 또는 §0)에만** 인문학 글귀를 **강렬한 한 줄(실제 출처 인용) + 쉬운 설명**으로 둔다.
일반 슬라이드엔 강렬한 한 줄을 만들지 않는다.

## 인용구 규칙

- **실제 출처가 있는 인용만** 따옴표로 쓴다. 따옴표 문장 바로 아래에 **저자·작품명·연도**를 표기한다.
- 전설·민담을 소재로 쓸 땐, 그 전설을 다룬 **실제 문헌의 한 줄**을 인용하고(예: 골렘 → 보르헤스 「골렘」),
  전설 자체의 근거는 '참고:'에 따로 둔다.
- **내가 압축·각색해 만든 문장을 실제 인용구인 양 따옴표로 제시하지 않는다.** 각색·요약은 설명에 녹인다.

## 원칙

1. **비유 먼저.** 추상 개념은 반드시 일상 사물/장면으로 1:1 치환해 설명한다.
2. **하나의 이야기로.** 정의 나열 금지 — 상황 → 문제 → 해결(그래서/그런데)로 흐르게.
3. **초보자 눈높이, 내용은 전문적.** 어려운 개념일수록 더 쉬운 비유로.
4. **내용이 사실인지 확인.** 그럴듯한 직관(흔한 오개념)을 사실처럼 쓰지 않는다. 메커니즘의 주체
   (모델 vs 앱)와 조건을 정확히 구분한다.
5. **세 화면은 한 흐름.** 슬라이드·대본·시뮬레이터는 같은 비유·같은 순서를 공유한다. **판서는 강사가
   시뮬레이터를 보고 직접 그린다 — 대본엔 '무엇을 그려라' 지시를 넣지 않는다(판서의 원본은 시뮬레이터).**
   시뮬레이터는 강의 끝에 복습용으로도 켠다.

## 톤

- 비난·훈계 금지. "이건 틀렸다"보다 "이렇게 보면 어때요?"
- 핵심 단어는 슬라이드에 다 박지 말고 강조로 남긴다.
