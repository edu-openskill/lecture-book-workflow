# 판서보드 스킬

펜·터치 기기에서 바로 여는 **단일 HTML 판서보드**를 만든다. 강의 슬라이드가 아니라, **직접 손으로 쓰고 그리는 칠판**이 본체다. 그래프·2차원 평면·필기·도형 설명에 쓴다.

## 산출물 (1개)
`<이름>_판서보드.html` — 단일 파일. 펜·터치 기기 브라우저에서 바로 열림. 외부 의존성 없음.

## 재현성 원칙 (중요)
HTML은 반드시 `template/board_template.html` 엔진을 **그대로 복사**해서 만든다.
엔진의 `<style>`/`<script>`(레이아웃·펜·칠판·격자·선택·이동·지우개·네비)는 **수정 금지**.
강의별로 바꾸는 곳은 단 두 군데뿐:
- `STEPS START ~ STEPS END` 사이의 `.step` 섹션 (슬라이드 화면. 빈 칠판만 쓸 거면 제목 한 컷만 둔다.)
- 필요 시 `:root`의 강조색 변수

## 만드는 법
1. `template/board_template.html`를 `<이름>_판서보드.html`로 복사한다.
2. `{{KICKER}}` 같은 플레이스홀더와 `STEPS` 영역을 채운다.
   - **빈 칠판 위주**로 쓸 거면: 제목 한 컷(`<section class="step active">`)만 남기고 나머지 `.step`은 지운다. 강의 중 `b`(판서모드)로 빈 칠판을 켠다.
   - **배경 화면이 필요**하면: 컷을 6~9개로 쪼개고 각 컷은 "한 줄 큰 메시지 + (선택) 보조 한 줄"만. 문장 3줄 이상 금지(나머지는 판서로 채운다).
3. 다른 파일은 만들지 않는다(이 스킬은 보드 하나만 낸다).

## 판서보드가 제공하는 도구 (엔진에 내장 — 직접 코딩 X)
- **펜**: 화면에 바로 판서. 색 5종 · 굵기 3종.
- **도형 스냅 (draw-and-hold)**: 직선·사각형·원/타원·삼각형을 그리고 펜을 댄 채 ~0.6초 멈추면 깔끔한 도형으로. 사각형은 두세 획으로 끊어 그려도 이어지면 합쳐 인식. 직선은 수평/수직(±10°)이면 축으로 교정. `s`로 끄고 켬(기본 켜짐).
- **🪧 판서모드 (b)**: 슬라이드를 가리고 초록 빈 칠판으로 전환. 다시 누르면 슬라이드로.
- **▦ 모눈 (g)**: 판서모드 칠판에 모눈종이 격자를 깔아 그래프·좌표평면 그리기 쉽게. (중심 십자선 없음)
- **✂ 선택**: 사각형으로 드래그해 그 안의 판서를 묶어 끌어 이동.
- **✋ 이동**: 그림을 누르면 **가장 가까운 한 획**만 집혀 이동. 손 안 떼고 그린 한 획 = 객체 하나라, 흰색·노란색이 겹쳐도 **하나만** 잡힌다. 커서가 손 모양(grab)으로 바뀐다.
- **지우개 (큰 네모)**: 한 변 72px짜리 큰 사각형으로, 지나간 부분의 획을 **실제로 잘라내 영구 삭제**(이동해도 복원 안 됨). 넓게 빨리 지운다.
- **선택한 획 삭제**: `✂ 선택`으로 묶은 직후 **지우개 버튼**(또는 `Delete`/`Backspace`)을 누르면 선택분이 삭제된다(`Ctrl+Z` 복구).
- **◇ 도형 버튼**: 도형 스냅 on/off(`s`).
- **되돌리기 (Ctrl+Z)** · **전체지움 (c)** · **Esc**(이동 중인 조각 고정).
- **← → / PageUp·Down**: 슬라이드 이동.
- **⛶ 전체화면 토글(좌상단)**: 갤럭시탭 등 F11 안 되는 태블릿용. 작은 코너 버튼만 눌러 전체화면(펜 판서 중 오작동 방지).

기술적으로 판서는 **벡터 획(stroke) 모델**이다(각 획 = 점들+색+굵기 객체). 그래서 겹친 획을 따로 선택·이동할 수 있고, 창 크기가 바뀌어도 다시 선명하게 그려진다. 자세한 내부 구조는 `reference/tools.md` 참고(단, 엔진은 수정하지 않는다).

## 검증
- 펜으로 흰색·노란색을 **교차**해 그린 뒤 ✋이동으로 **하나만** 잡히는지.
- 한 획의 중간을 지우개로 지운 뒤, 그 획을 이동했을 때 **복원되지 않는지**.
- 100% 화면에서 하단 툴바 버튼 라벨이 **한 줄**로 나오는지.
- 모눈(g)·판서모드(b)·전체화면(⛶) 동작.

## CLAUDE.md 규칙
프로젝트에 `CLAUDE.md`가 있으면 톤·색·용어 규칙을 따른다. 모든 판서보드에는 좌상단 전체화면 토글 버튼(⛶)이 반드시 있어야 한다(엔진에 이미 포함됨).
