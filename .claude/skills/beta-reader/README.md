# Beta Reader — 설계 문서

출판 업계의 "베타 리더"를 AI로 자동화한 스킬.
한 명이 아니라 **10명이 동시에** 읽고 각자 관점에서 피드백한다.

## 왜 만들었나

- 저자 혼자 읽으면 자기 글의 문제를 못 본다
- 한 명이 리뷰하면 그 사람의 배경에 편향된다
- 10명의 다양한 배경이 동시에 읽으면 **특정 수준에만 맞춰진 챕터**가 바로 드러난다

## 설계 흐름

```
0. 프로젝트 감지
   projects/ 아래 최근 수정 폴더에서 seed.md, outline, chapters, code 자동 탐색
   경로 하드코딩 없음 → 어떤 책이든 동작

1. 페르소나 자동 생성
   seed.md에서 "책 주제" + "독자 상수" 읽기
   → 10명 페르소나 생성 (고정 목록 아님, 주제에 맞게 매번 새로 생성)

2. 컨텍스트 로드
   챕터 전문 + 실습 코드 + seed 의도를 각 페르소나에게 전달

3. 10명 병렬 디스패치
   Agent 도구로 10명 동시 실행 → 각자 독립 평가

4. 평가 (체크리스트 4개 영역)
   이야기 / 기술 / 실습 / 전체

5. 종합 리포트 생성
   review/beta-reader-CH{N}.md에 저장
```

## 페르소나 분배 규칙

10명을 아래 비율로 생성한다. seed.md의 책 주제에 맞게 구체적인 배경을 채운다.

| 유형 | 인원 | 예시 (RAG 책) | 왜 이 비율인가 |
|------|------|-------------|--------------|
| 주제와 가까운 배경 | 2명 | 데이터 분석가, ML 엔지니어 | 기술적 정확성 검증 |
| 주제와 먼 배경 | 2명 | 프론트엔드, SI 개발자 | 설명이 충분한지 검증 |
| 비개발 배경 | 2명 | PM/기획자, 비전공 전직자 | 용어 장벽, 접근성 검증 |
| 숙련자 | 2명 | CTO/리드, 시니어 아키텍트 | 아키텍처/확장성/실무 관점 |
| 초보 | 1명 | 부트캠프 수료 | 최소 수준에서 따라갈 수 있는지 |
| 학생 | 1명 | CS 전공 3학년 | 이론↔실무 연결 검증 |

## 평가 체크리스트

### 이야기 파트 — 글의 품질

| 항목 | 보는 것 |
|------|--------|
| 몰입 | 빠져드는가, 지루한 구간은 |
| 비유 | 내 배경에서 와닿는가, 억지스러운 비유는 |
| 톤 | AI스러운 문장("이것이야말로", "기억하십니까?") |
| 용어 도입 | 갑자기 나오지 않는가, 비유→정의 순서 |
| 등장인물 | 대화가 자연스러운가 |

### 기술 파트 — 학습 효과

| 항목 | 보는 것 |
|------|--------|
| 용어 연결 | 이야기 비유 → 기술 정의 연결 |
| 코드 설명 | 내 수준에서 이해 가능한가 |
| 데이터 시각화 | 추상 설명에 실제 데이터(JSON/출력/다이어그램)가 있는가 |
| 파일 트리 | [실습]/[설명]/[참고] 분류가 명확한가 |

### 실습 — 실제로 따라할 수 있는가

| 항목 | 보는 것 |
|------|--------|
| 환경 구축 | pip, venv, Docker에서 막히는 지점 |
| 실행 결과 | 챕터 설명과 실제 결과 일치 |
| TODO | 힌트가 충분한가 |
| 에러 대응 | 에러 시 해결법이 챕터에 있는가 |

### 전체 — 책의 흐름

| 항목 | 보는 것 |
|------|--------|
| 학습 곡선 | 이전 챕터 대비 난이도 점프 |
| 분량 | 길다/적당하다/짧다 |
| 다음 예고 | 흥미 유발 여부 |

## 리포트 구조

```
요약 (N/10 통과)
  ↓
공통 피드백 (3명 이상 동일 의견 = 진짜 문제)
  ↓
심각도별 이슈 (높음/중간/낮음)
  ↓
페르소나별 상세 (각자 이야기/기술/실습 점수 5점 만점)
  ↓
수정 제안 (우선순위순, 위치+라인 포함)
```

### 핵심: "3명 이상 동일 의견" 규칙

1명만 지적한 건 그 페르소나의 특수 사정일 수 있다.
**3명 이상**이 같은 걸 지적하면 챕터 자체의 문제다.
이걸 "공통 피드백"으로 올려서 저자가 먼저 보게 한다.

## 범용성

| 항목 | 설계 |
|------|------|
| 프로젝트 경로 | `projects/` 아래 자동 감지, 하드코딩 없음 |
| 페르소나 | seed.md 주제에 맞게 매번 새로 생성 |
| 코드 없는 챕터 | 실습 평가 건너뛰고 이야기+기술만 |
| 이전 리포트 | 있으면 "지적 사항이 개선되었는가" 확인 |
| 등장인물 | 있는 책만 등장인물 평가 포함 |

## 사용

```bash
/beta-reader CH04          # 단일 챕터
/beta-reader CH01-CH03     # 범위 (순차 실행)
```

리포트 저장 위치: `{프로젝트}/review/beta-reader-CH{N}.md`
