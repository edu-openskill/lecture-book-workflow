# 챕터 포맷 규칙

실습 중심 챕터(CH01, CH04~CH11)의 기술 파트 **표준 포맷**. `chapters/**` 파일에 적용.

캡션·그림번호 규칙은 `structure.md`, 코드블록 상세는 `code.md` 참조.

---

## 챕터 전체 구조

```
:::goal          — "이번 챕터가 끝나면" 3~4줄
::::prep         — 폴더 이동 / 파일 트리 / 환경 구축 / 라이브러리 / 실습 순서
본문 섹션        — N.1 ~ N.M (아래 "실습 섹션 포맷" 반복)
용어 정리        — 비유·진짜용어·정식정의 3열 표
이것만은 기억하자 — 3줄 + 다음 챕터 예고
```

---

## 실습 섹션 포맷 (한 실험 단위)

```
1. 개념 설명        — 비유 + 이론 (이야기 톤)
2. 개념 다이어그램   — HTML inline, "그림 N-N." 접두어, text-align:center
3. 연결 문장        — "이 흐름을 코드로 구현합니다" (다이어그램→코드 전환)
4. 파일 경로 + TODO — "`파일경로`를 열고 TODO의 `pass`를 지우고 아래 코드를 작성합니다"
5. 코드블록         — [실습 N] 태그, 내부에 # TODO: + # 1. 2. 단계
6. 코드 후 설명     — 매개변수 역할, 보조 함수 테이블 (2개 이상이면)
7. bash 실행        — ```bash [터미널] 실험 N-N 실행. 제목
8. 결과 다이어그램   — HTML inline, "그림 N-N." 접두어, 실제 실행 값
9. 결과 해석        — 1~2문장으로 숫자의 의미 풀이
10. 커스텀 bash     — (있으면) "다른 질문으로 실험해 보세요"
11. :::tip          — "실험에서 생각해 볼 것들" (볼드 키워드: 설명 형식)
12. 스토리 전환     — 캐릭터 대사·내면독백 → 다음 실험의 문제 제기
```

## 각 요소 상세

### 개념 다이어그램

- HTML `<div>` inline 스타일. CSS 변수(`var(--color-*)`, `var(--fs-*)`) 사용
- 제목·캡션 형식은 `structure.md` 참조

### 코드블록 앞 안내

`code.md`의 "[실습] 코드블록 앞 필수 안내" 참조.

### 코드블록

`code.md`의 "실습 코드블록 작성 규칙" 참조. 핵심:
- `[실습 N]` 태그 + 파일 경로 + 제목
- 내부에 `# TODO:` + `# N.` 단계 번호

### 코드 후 설명

- 핵심 동작 1~2문장
- 매개변수 3개 이상이면 테이블
- 보조 함수 2개 이상이면 테이블 (함수명 · 역할)
- `embeddings`, `documents` 같은 외부 주입 매개변수는 출처 명시

### 결과 다이어그램

- HTML inline, `font-family:var(--font-mono)` 터미널 스타일
- **실제 실행 결과 값** 사용 (더미 금지)
- 제목: `"그림 N-N. 실험 M-K. 제목"`
- 하단 주석: 핵심 포인트 한 줄 (`color:var(--color-success-text)`)

### 결과 해석

```markdown
직접 검색에서는 1위(0.5917)와 3위(0.4455) 사이 점수 차이가 0.15에 불과합니다.
HyDE에서는 가상 답변이 "연차유급휴가"라는 단어를 직접 포함하기 때문에 유사도가 전반적으로 올랐습니다.
```

- 숫자를 인용하며 의미를 풀어줌
- "높아졌습니다" 같은 추상적 서술 금지 → 구체적 수치 비교

### :::tip 블록

```markdown
:::tip
**HyDE 실험에서 생각해 볼 것들**

- **유사도 점수 차이**: 설명...
- **가상 답변의 품질**: 설명...
- **순위 변화**: 설명...
:::
```

- 제목: "**{실험명} 실험에서 생각해 볼 것들**"
- 각 항목: `**볼드 키워드**: 설명` 형식
- CH08의 "리랭킹 실험에서 생각해 볼 것들" 패턴 참조

### 스토리 전환

```markdown
*됐다.* 오픈이가 터미널을 내려다봤습니다. ...

"다른 질문을 넣어 봤습니다. ..." → 다음 실험의 문제 제기
```

- 캐릭터 물리적 행동 + 내면독백(`*이탤릭*`)
- "이걸 풀었더니 저 문제" 가 아니라 "이 도구는 이런 상황에서 효과적. 다른 상황은?" 톤
- 감각 묘사 1개 이상 (`storytelling.md` 참조)
