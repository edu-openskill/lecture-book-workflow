# 글로벌 코드 규칙

모든 에이전트가 준수하는 코드 표시 규칙.

---

## 코드 블록

- 모든 코드 블록. 언어 태그 필수 (`java`, `python`, `bash` 등)
- 구조. 목적 한 줄 → 코드 → 설명 → 실행 결과
- 핵심만 10~15줄 (전체는 versions/ 참조)
- bash/shell. 상세 설명 불필요
- 번호 주석(①②③) 금지
- 코드 내 단계 주석. `# N. 설명` 형태로 코드 윗줄에 작성. 학생이 각 줄의 역할을 알 수 있게 한다
- 실습 코드블록 첫 줄. `# 파일명 — TODO: 예제 레포에 있는 TODO 내용 그대로` 형태
- 코드블록 아래 설명. 각 코드블록 뒤에 핵심 동작을 1~2문장으로 설명

## 파트별 코드 규칙

- 이야기 파트에 코드 블록 절대 없음
- 기술 파트에서만 코드 등장
- code/는 원천 소스코드 (수정하지 않음). versions/는 code/에서 챕터별로 분해한 성장 예제
- 챕터의 코드 블록은 해당 versions/ 코드와 일치해야 함. 불일치 시 챕터를 versions/에 맞춤

## 코드 파일 유형

- [실습] 파일. 예제 레포에 TODO+pass로 제공. 챕터의 코드블록을 보고 pass를 지우고 작성. 안내 문구: "TODO의 `pass`를 지우고 아래 코드를 작성합니다"
- [설명] 파일. 핵심 흐름만 발췌. 전체 코드는 완성본 레포(ai-qa-lag) 참조
- [참고] 파일. 코드 블록 없음. 메서드 표로 요약

## Git 레포 구조

- **ai-qa-lag**: 완성본 레포. 전체 동작 코드. 복사 붙여넣기 또는 정답 확인용
- **ai-qa-lag-ex**: 예제 레포. 디렉토리 구조 + 빈 파일/스켈레톤. 독자가 클론 → 챕터 보면서 코드 작성
- 실습 흐름: ai-qa-lag-ex 클론 → 챕터 따라 파일 생성/작성 → 막히면 ai-qa-lag 참고

## 실험/실습 스크립트 출력

- 터미널 출력에 장식선 금지 (`console.rule()`, `───` 등의 장식 구분선 사용 금지)
- 제목은 `console.print("[bold]제목[/bold]")` 으로 간결하게 표시
- Rich Table의 `title` 파라미터로 섹션 구분 (장식선 대신)
- 모든 스크립트 출력은 Rich Console을 통해 출력 (print 대신 console.print)
- 캡처 워크플로우: 스크립트가 Rich Console로 출력 → `book_capture.py --cmd`로 캡처
- 캡처 실행 전에 대상 스크립트에 `console.rule()` 잔존 여부를 grep으로 점검하라 → why-log.md#2026-03-15-26

## 스킬 호출

- 완성 코드 분석 시 → `code` 스킬 호출
- 코드 설명 패턴 적용 시 → `code` 스킬 호출
- 다이어그램 생성 시 → `visual` 스킬 호출

## 코드 일치 점검

versions/ 코드가 변경되면 챕터 기술 파트가 불일치할 수 있다.

### writer 점검
- `chapters/*.md`의 코드 블록이 해당 `versions/` 코드와 일치하는지 확인
- 불일치 시 챕터를 versions/에 맞춤

### editor 점검
- `chapters/*.md`의 코드 블록과 `versions/` 사이 불일치 감지
- 함수명, 파라미터, 임포트 경로 변경 여부 확인
- 불일치 발견 시 경고 + 수정 대상 챕터 목록 제시
